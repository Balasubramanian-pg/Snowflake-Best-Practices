# Assigning a Resource Monitor to a Warehouse using ALTER WAREHOUSE

**Description:**
This topic explains how to assign a resource monitor to a warehouse in Snowflake using the ALTER WAREHOUSE command. This allows administrators to monitor and control warehouse usage. Resource monitors help track credits used and can automatically suspend or terminate warehouses to manage costs.

## Short Explanation
**Overview:** Assign a resource monitor to a warehouse to track usage and manage costs.

**Problem Solved:** Uncontrolled warehouse usage leading to unexpected costs.

**Importance:** Essential for cost management and optimizing warehouse usage in Snowflake.

**Use Cases:**
- Assigning monitors to warehouses with high usage
- Automatically suspending warehouses during off-peak hours

### Typical Scenario
A company wants to monitor and control the usage of their Snowflake warehouses to avoid unexpected costs. They create a resource monitor and assign it to their warehouses.

### Core Snowflake SQL Objects Used
`ALTER WAREHOUSE`, `CREATE RESOURCE MONITOR`

### Code Source Execution
```sql
ALTER WAREHOUSE my_warehouse SET RESOURCE_MONITOR = my_resource_monitor;
```

### Example Execution Logic Step-by-Step
To assign a resource monitor to a warehouse, use the ALTER WAREHOUSE command with the RESOURCE_MONITOR option. First, create a resource monitor using CREATE RESOURCE MONITOR. Then, use the ALTER WAREHOUSE command to assign the monitor to a warehouse.

### Implementation Example
CREATE RESOURCE MONITOR my_resource_monitor WITH CREDIT QUOTA = 100;
ALTER WAREHOUSE my_warehouse SET RESOURCE_MONITOR = my_resource_monitor;

### Explanation of the Code
- Create a resource monitor with a specified credit quota.
- Assign the resource monitor to a warehouse using ALTER WAREHOUSE.

### When to Use
- When you want to monitor and control warehouse usage
- When you need to automatically suspend or terminate warehouses

### When Not to Use
- When you don't need to monitor warehouse usage
- When you have a different method for managing costs

### Common Mistakes
- Forgetting to create the resource monitor before assigning it
- Not specifying the correct credit quota
