# Conditional Task Execution using WHEN Clauses in Snowflake

**Description:**
This topic explains how to execute tasks conditionally in Snowflake using WHEN clauses. It allows for flexible task execution based on specific conditions, enabling more efficient and dynamic data processing pipelines.

## Short Explanation
**Overview:** Conditional task execution in Snowflake enables tasks to run based on specific conditions.

**Problem Solved:** It solves the problem of needing to control task execution based on dynamic conditions.

**Importance:** This feature is crucial in analytics and warehousing for creating flexible and efficient data processing workflows.

**Use Cases:**
- Data quality checks
- Dynamic data loading
- Conditional data transformation

### Typical Scenario
A common scenario is a data pipeline that loads data into a staging table and then transforms it only if certain conditions are met, such as data quality checks passing.

### Core Snowflake SQL Objects Used
`CREATE TASK`

### Code Source Execution
```sql
CREATE TASK my_task
  WAREHOUSE = 'my_warehouse'
  SCHEDULE = '0 12 * * *'
  COMMENT = 'Task to load and transform data'
AS
  CALL my_stored_procedure();

ALTER TASK my_task
  ADD WHEN
    CONDITION = my_condition;
```

### Example Execution Logic Step-by-Step
In this example, a task 'my_task' is created to execute a stored procedure 'my_stored_procedure' on a scheduled basis. The task is then altered to include a conditional execution using the 'WHEN' clause, which checks a condition 'my_condition' before executing the task.
