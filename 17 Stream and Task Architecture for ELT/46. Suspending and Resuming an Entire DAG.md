# Suspending and Resuming an Entire DAG in Snowflake

**Description:**
This topic explains how to suspend and resume an entire Directed Acyclic Graph (DAG) in Snowflake, which is a collection of tasks that are executed in a specific order.

## Short Explanation
**Overview:** Suspending and resuming a DAG allows for efficient management of complex task workflows in Snowflake.

**Problem Solved:** It solves the problem of needing to pause or restart a complex task workflow due to maintenance, errors, or other issues.

**Importance:** This feature is important in analytics and warehousing as it enables efficient management of ETL/ELT workflows.

**Use Cases:**
- Pausing a DAG during maintenance window
- Resuming a failed DAG after error resolution

### Typical Scenario
A data engineer needs to pause a complex ETL workflow during a maintenance window and resume it once the maintenance is complete.

### Core Snowflake SQL Objects Used
`ALTER TASK`, `SHOW TASKS`

### Code Source Execution
```sql
ALTER TASK my_task SUSPEND;
```

### Example Execution Logic Step-by-Step
To suspend an entire DAG, use the ALTER TASK command with the SUSPEND option. To resume, use the RESUME option.

### Implementation Example
CREATE TASK my_task
  WAREHOUSE = 'my_warehouse'
  SCHEDULE = '0 1 * * *'
  COMMENT = 'My task';

ALTER TASK my_task SUSPEND;

-- Resume the task when ready
ALTER TASK my_task RESUME;

### Explanation of the Code
- Step 1: Create a task with a specific schedule and warehouse
- Step 2: Suspend the task using ALTER TASK with SUSPEND option
- Step 3: Resume the task using ALTER TASK with RESUME option

### When to Use
- During maintenance windows
- When a task fails and needs to be restarted

### When Not to Use
- When a task is part of a critical workflow that cannot be paused

### Common Mistakes
- Forgetting to resume a suspended task
- Suspending a task that is not part of a DAG

**Description:** The output of SHOW TASKS command will indicate the status of the task (suspended or resumed)
