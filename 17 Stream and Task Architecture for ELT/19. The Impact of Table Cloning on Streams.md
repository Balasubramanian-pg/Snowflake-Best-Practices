### Example Execution Logic Step-by-Step
When a table is cloned, the stream is not automatically cloned. The stream continues to capture changes from the original table. If the cloned table is used as a new source, a new stream must be created.

### Implementation Example
CREATE TABLE orders (id INT, name STRING); INSERT INTO orders VALUES (1, 'John'); CREATE STREAM orders_stream ON TABLE orders; CREATE TABLE orders_clone CLONE orders; -- orders_stream still captures changes from orders, not orders_clone

### Explanation of the Code
- Create a table and a stream on that table.
- Clone the table to create a new, independent copy.
- The stream continues to capture changes from the original table, not the cloned table.
