# Dynamic Data Masking in Snowflake

**Description:**
Dynamic Data Masking (DDM) is a security feature in Snowflake that allows administrators to limit sensitive data exposure by masking it to unauthorized users. DDM applies masks to data at query time, ensuring that users only see authorized data. This feature helps protect sensitive data while maintaining data usability.

## Short Explanation
**Overview:** Dynamic Data Masking (DDM) is a Snowflake feature that masks sensitive data at query time.

**Problem Solved:** DDM solves the problem of sensitive data exposure to unauthorized users.

**Importance:** DDM is crucial in analytics and warehousing for maintaining data security and compliance.

**Use Cases:**
- Restricting access to personally identifiable information (PII) for analysts\n- Masking financial data for junior employees

### Typical Scenario
A marketing analyst needs to access customer data for campaign analysis, but should not see sensitive information like social security numbers. DDM can mask this sensitive data for the analyst's role.

`CREATE MASKING POLICY`, `ALTER TABLE ... ADD MASKING POLICY`

To implement DDM, first create a masking policy using CREATE MASKING POLICY. Then, apply this policy to a table column using ALTER TABLE ... ADD MASKING POLICY. When a user queries the data, Snowflake applies the mask based on the user's role.
