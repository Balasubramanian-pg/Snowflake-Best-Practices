# Appending Timestamps for Point-in-Time Clones in Snowflake

**Description:**
This Snowflake best practice explains how to create point-in-time clones of databases or schemas by appending timestamps to their names. This allows for easy tracking and management of cloned environments. By appending timestamps, you can ensure that each clone is uniquely named and easily identifiable.

### Example Execution Logic Step-by-Step
To create a point-in-time clone of a database or schema, use the CREATE ... CLONE statement and append a timestamp to the clone's name. This can be achieved using Snowflake's SYSTEM$GENERATE_UUID() function or a SQL query to get the current timestamp.

### Implementation Example
CREATE OR REPLACE PROCEDURE create_clone_with_timestamp()
  RETURNS STRING
  LANGUAGE SQL
  EXECUTE AS CALLER
  BEGIN
    DECLARE 
      timestamp TIMESTAMP = CURRENT_TIMESTAMP;
      clone_name STRING = 'mydatabase_clone_' || TO_VARCHAR(timestamp, 'YYYY_MM_DD_HH24_MM_SS');
    CREATE DATABASE ${clone_name} CLONE mydatabase;
    RETURN ${clone_name};
  END;

### Explanation of the Code
- Step 1: Get the current timestamp using Snowflake's CURRENT_TIMESTAMP function.
- Step 2: Construct the clone name by appending the timestamp to the original database or schema name.
- Step 3: Use the CREATE ... CLONE statement to create the clone with the generated name.
