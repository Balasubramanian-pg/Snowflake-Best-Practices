# Cross-Database Cataloging using Fully Qualified Names in Snowflake

**Description:**
This topic explains how to catalog data across multiple databases in Snowflake using fully qualified names. It enables users to access and manage metadata across different databases, promoting data consistency and reducing complexity. By using fully qualified names, users can uniquely identify database objects, such as tables and views, across multiple databases.

## Short Explanation
**Overview:** Cross-database cataloging allows users to access metadata from multiple databases using a single query.

**Problem Solved:** It solves the problem of managing and accessing metadata across multiple databases, reducing complexity and improving data consistency.

**Importance:** This feature is important in analytics and warehousing as it enables users to gain a unified view of their data across multiple databases.

**Use Cases:**
- Data warehousing
- Data integration
- Metadata management

### Typical Scenario
A company has multiple databases, each containing different sets of data. They want to create a unified view of their data across all databases and manage metadata centrally.

### Core Snowflake SQL Objects Used
`INFORMATION_SCHEMA`

### Code Source Execution
```sql
SELECT * FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_SCHEMA = 'schema_name' AND TABLE_NAME = 'table_name';
```

### Example Execution Logic Step-by-Step
To access metadata from multiple databases, use fully qualified names in the format 'database_name.schema_name.table_name'. For example: SELECT * FROM mydatabase.myschema.mytable;

### Implementation Example
CREATE VIEW mycrossdbview AS SELECT * FROM mydatabase1.myschema.mytable UNION ALL SELECT * FROM mydatabase2.myschema.mytable;

### Explanation of the Code
- Step 1: Identify the databases and schemas that contain the metadata you want to access.
- Step 2: Use fully qualified names to query the metadata from each database.

### When to Use
- When managing metadata across multiple databases
- When creating a unified view of data across multiple databases

### When Not to Use
- When working with a single database
- When using Snowflake's data sharing feature

### Common Mistakes
- Using unqualified names, leading to ambiguity and errors
- Not specifying the correct database and schema names
