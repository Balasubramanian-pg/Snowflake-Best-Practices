# Dynamic Data Masking in Snowflake

**Description:**
Dynamic Data Masking (DDM) is a security feature in Snowflake that allows administrators to limit sensitive data exposure by masking it to unauthorized users. DDM ensures that sensitive data is not accidentally exposed to users who do not have the necessary privileges to view it. This feature helps organizations to meet compliance requirements and protect sensitive data. DDM can be applied to specific columns in a table.

## Short Explanation
**Overview:** Dynamic Data Masking is a feature that masks sensitive data at query time.

**Problem Solved:** Unauthorized access to sensitive data.

**Importance:** Protects sensitive data and meets compliance requirements.

**Use Cases:**
- Securing customer PII data\n- Protecting financial data

### Typical Scenario
A company wants to expose customer data to analysts, but the data contains sensitive information such as social security numbers and credit card numbers. The company uses DDM to mask this sensitive data, ensuring that only authorized users can view it.

### Core Snowflake SQL Objects Used
`CREATE MASKING POLICY`, `ALTER TABLE`

### Code Source Execution
```sql
CREATE MASKING POLICY mask_ssn_policy FOR COLUMN customers.ssn USING (CASE WHEN CURRENT_ROLE() IN ('ANALYST') THEN ssn ELSE 'XXXX-XXX-' || SUBSTR(ssn, -4) END); ALTER TABLE customers ADD MASKING POLICY mask_ssn_policy;
```

### Example Execution Logic Step-by-Step
To implement DDM, first create a masking policy using the CREATE MASKING POLICY statement. This policy defines how the data should be masked based on the user's role. Then, apply the policy to the relevant column(s) in a table using the ALTER TABLE statement.
