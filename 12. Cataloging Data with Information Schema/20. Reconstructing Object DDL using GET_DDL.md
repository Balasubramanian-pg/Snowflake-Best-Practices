# Reconstructing Object DDL using GET_DDL in Snowflake

**Description:**
The GET_DDL function in Snowflake allows users to reconstruct the Data Definition Language (DDL) statements used to create objects such as tables, views, and procedures. This feature is useful for documenting and versioning database objects. By using GET_DDL, users can retrieve the exact DDL statement used to create an object, including the object's properties and characteristics.

## Short Explanation
**Overview:** GET_DDL function retrieves the DDL statement used to create a Snowflake object.

**Problem Solved:** Provides a way to document and version database objects.

**Importance:** Essential for maintaining data lineage and object metadata.

**Use Cases:**
- Documenting database objects
- Versioning object changes
- Recreating objects in another account

### Typical Scenario
A database administrator wants to document all objects in a Snowflake database and store the DDL statements in a version control system.

### Core Snowflake SQL Objects Used
`GET_DDL`

### Code Source Execution
```sql
SELECT GET_DDL('table', 'my_table');
```

### Example Execution Logic Step-by-Step
The GET_DDL function takes two arguments: the object type and the object name. In this example, the function retrieves the DDL statement used to create the 'my_table' table.

### Implementation Example
To reconstruct the DDL for all tables in a schema:
SELECT GET_DDL('table', table_name) AS ddl
FROM information_schema.tables
WHERE table_schema = 'my_schema';

### Explanation of the Code
- Step 1: Use the INFORMATION_SCHEMA to retrieve a list of tables in the schema.
- Step 2: Use the GET_DDL function to retrieve the DDL statement for each table.

### When Not to Use
