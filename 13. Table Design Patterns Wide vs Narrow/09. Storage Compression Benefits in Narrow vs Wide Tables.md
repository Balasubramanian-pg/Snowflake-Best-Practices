A data warehousing scenario where a large dataset needs to be stored and queried efficiently. The dataset has many columns, but not all columns are frequently queried.

### Example Execution Logic Step-by-Step
In Snowflake, narrow tables with fewer columns tend to compress better than wide tables with many columns. This is because Snowflake compresses data based on the number of distinct values in each column. With fewer columns, there are more rows to compress, resulting in better compression ratios.

### Implementation Example

 id INT,
 name VARCHAR(100),
 email VARCHAR(100),
 phone VARCHAR(20),
 address VARCHAR(200),
 city VARCHAR(50),
 state VARCHAR(50),
 zip VARCHAR(10)
);

CREATE TABLE narrow_table (id INT, name VARCHAR(100), email VARCHAR(100));

### Explanation of the Code
- Step 1: Create a wide table with many columns, which can lead to poor compression ratios.
- Step 2: Create a narrow table with fewer columns, which can result in better compression ratios.

### When to Use
- Use narrow tables for large datasets with many rows
- Use narrow tables for columns with high cardinality

### When Not to Use
- Avoid wide tables with many columns and low cardinality
- Avoid using wide tables for large datasets

### Common Mistakes
- Designing wide tables with low cardinality columns
- Not considering data compression when designing tables
