# Difference Between Information Schema Views and Table Functions in Snowflake

**Description:**
This topic explains the distinction between Information Schema views and Table Functions in Snowflake, highlighting their usage and benefits. ### Typical Scenario
A data analyst needs to retrieve metadata about tables in a Snowflake database. They can use Information Schema views like 'INFORMATION_SCHEMA.TABLES' to get a list of tables, or use a Table Function like 'TABLE(INFORMATION_SCHEMA.TABLES())' to achieve similar results.

### Example Execution Logic Step-by-Step
Information Schema views are system views that provide metadata about database objects, such as tables, columns, and data types. Table Functions, on the other hand, are user-defined functions that return a table as output. In Snowflake, Table Functions can be used to query data in a more flexible way than Information Schema views.

### Implementation Example
CREATE FUNCTION get_table_info() RETURNS TABLE (table_name STRING, table_schema STRING) AS 'SELECT table_name, table_schema FROM INFORMATION_SCHEMA.TABLES;'; SELECT * FROM TABLE(get_table_info());

### Explanation of the Code
- Step 1: Create a Table Function 'get_table_info' that returns a table with 'table_name' and 'table_schema' columns.
- Step 2: Query the Table Function using 'SELECT * FROM TABLE(get_table_info());' to retrieve the desired metadata.

### When to Use
- Use Information Schema views when you need to access standardized metadata about database objects.
- Use Table Functions when you need more flexibility in querying data or require custom metadata retrieval.

### When Not to Use
- Avoid using Information Schema views when you need to perform complex queries or require custom data processing.
- Avoid using Table Functions when you need simple, standardized metadata access.
