# Case Sensitivity and Upper Case Matching in Information Schema Queries

**Description:**
This topic explains how case sensitivity and upper case matching work in Snowflake's Information Schema queries. It provides guidance on querying Information Schema objects and columns while considering case sensitivity. Proper handling of case sensitivity ensures accurate results and avoids errors.

## Short Explanation
**Overview:** Snowflake's Information Schema is case-sensitive, and column names are typically returned in upper case.

**Problem Solved:** Handling case sensitivity when querying Information Schema objects and columns.

**Importance:** Ensures accurate results and avoids errors in data cataloging and metadata queries.

**Use Cases:**
- Querying table columns
- Checking schema objects

### Typical Scenario
A data analyst needs to retrieve a list of columns for a specific table using the Information Schema. The table name and column names are in mixed case, but the analyst knows that Snowflake converts them to upper case.

### Core Snowflake SQL Objects Used
`INFORMATION_SCHEMA.COLUMNS`

### Code Source Execution
```sql
SELECT * FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_NAME = 'my_table';
```

### Example Execution Logic Step-by-Step
In Snowflake, Information Schema queries are case-sensitive. When querying table columns, the TABLE_NAME and COLUMN_NAME columns are typically in upper case. To retrieve columns for a table named 'my_table', the query should use the upper case name.

### Implementation Example
To get columns for a table 'my_table', use the following query:
SELECT COLUMN_NAME, DATA_TYPE
FROM INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_NAME = 'MY_TABLE';

### Explanation of the Code
- The query selects COLUMN_NAME and DATA_TYPE from the INFORMATION_SCHEMA.COLUMNS view.
- The WHERE clause filters results for the table with the name 'MY_TABLE', which matches the table 'my_table' due to case sensitivity.
