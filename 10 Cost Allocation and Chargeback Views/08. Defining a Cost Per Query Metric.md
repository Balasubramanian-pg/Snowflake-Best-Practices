### Core Snowflake SQL Objects Used
`CREATE VIEW`

### Code Source Execution
```sql
CREATE VIEW cost_per_query AS
SELECT 
  query_id,
  query_text,
  execution_time,
  credits_used,
  credits_used / execution_time AS cost_per_second
FROM 
  TABLE(INFORMATION_SCHEMA.QUERY_HISTORY_BY_SESSION());
```

### Example Execution Logic Step-by-Step
The SQL code creates a view called `cost_per_query` that calculates the cost per second for each query executed in the current session. The view uses the `QUERY_HISTORY_BY_SESSION` table function to retrieve query history and calculates the cost per second by dividing the credits used by the execution time.
