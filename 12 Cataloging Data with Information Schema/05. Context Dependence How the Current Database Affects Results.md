### Example Execution Logic Step-by-Step
In Snowflake, the current database and schema determine the context for operations. For example, when you run a query without specifying the database and schema, Snowflake uses the current database and schema. To avoid ambiguity, you can use fully qualified object names or switch to the desired database and schema using the USE statements.

### Implementation Example
To manage context dependence, use the USE DATABASE and USE SCHEMA statements to switch between databases and schemas. For instance:

USE DATABASE sales;
USE SCHEMA usa;
SELECT * FROM customers;

### Explanation of the Code
- Step 1: Switch to the 'sales' database using the USE DATABASE statement.
- Step 2: Switch to the 'usa' schema using the USE SCHEMA statement.
- Step 3: Run a query on the 'customers' table in the current schema.
