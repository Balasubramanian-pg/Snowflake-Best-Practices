# CREATE RESOURCE MONITOR Syntax Overview

**Description:**
This topic provides an overview of the CREATE RESOURCE MONITOR syntax in Snowflake, which allows you to create a resource monitor to track and manage your Snowflake credits usage.

## Short Explanation
**Overview:** The CREATE RESOURCE MONITOR statement creates a resource monitor that tracks and manages credits usage for a specified set of warehouses, databases, roles, or users.

**Problem Solved:** It helps organizations monitor and control their Snowflake credits usage, preventing unexpected costs and optimizing resource utilization.

**Importance:** It matters in analytics and warehousing as it enables organizations to manage their Snowflake costs and resources effectively.

**Use Cases:**
- Monitoring credits usage for a specific warehouse
- Managing credits usage for multiple databases

### Typical Scenario
A business wants to monitor and manage credits usage for their development and production environments separately.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR my_resource_monitor
CREDITS_QUOTA = 100
TRIGGERED ALERTS = 80, 90
NOTIFICATION_INTEGRATION = 'my_notification_integration';
```

### Example Execution Logic Step-by-Step
The CREATE RESOURCE MONITOR statement creates a resource monitor named 'my_resource_monitor' with a credits quota of 100. It also sets triggered alerts at 80% and 90% of the quota and specifies a notification integration.

### Implementation Example
CREATE RESOURCE MONITOR dev_resource_monitor
FOR WAREHOUSE = 'dev_warehouse'
CREDITS_QUOTA = 50
TRIGGERED ALERTS = 40, 50
NOTIFICATION_INTEGRATION = 'dev_notification_integration';

### Explanation of the Code
- Step 1: Create a resource monitor named 'dev_resource_monitor' for the 'dev_warehouse' warehouse.
- Step 2: Set the credits quota to 50 and triggered alerts at 40% and 50% of the quota.
- Step 3: Specify a notification integration named 'dev_notification_integration'.

### When to Use
- When you want to monitor credits usage for a specific warehouse or set of warehouses
- When you want to manage credits usage for multiple databases or roles

### When Not to Use
- When you have a single, simple Snowflake deployment with minimal credits usage
- When you are not concerned about optimizing credits usage

- Not specifying the correct credits quota or notification integration
