# Renaming Databases and Schemas in Snowflake using ALTER Statements

**Description:**
This topic explains how to rename databases and schemas in Snowflake using ALTER statements. Renaming these objects can help with organization, clarity, and adherence to naming conventions. Snowflake provides specific commands to modify the names of databases and schemas.

## Short Explanation
**Overview:** Snowflake allows renaming databases and schemas using the ALTER statements.

**Problem Solved:** Renaming databases and schemas helps maintain an organized structure and ensures that names are descriptive and follow conventions.

**Importance:** Proper naming conventions are crucial for large databases and complex data warehouses for better management and understanding.

**Use Cases:**
- Renaming a database to reflect a change in company name or product line.
- Renaming a schema to better organize objects within a database.

### Typical Scenario
A company decides to rebrand and needs to rename their database and several schemas to reflect the new brand name. They use Snowflake's ALTER statements to accomplish this task.

### Core Snowflake SQL Objects Used
`ALTER DATABASE`, `ALTER SCHEMA`

### Code Source Execution
```sql
ALTER DATABASE my_old_database RENAME TO my_new_database;
```

### Example Execution Logic Step-by-Step
To rename a database or schema, you use the ALTER statement followed by the current name of the object, the RENAME TO clause, and then the new name. For example, to rename a database from 'my_old_database' to 'my_new_database', you would use the command: ALTER DATABASE my_old_database RENAME TO my_new_database;

### Implementation Example
Suppose we have a database named 'sales_db' and we want to rename it to 'business_sales'. The SQL command would be: ALTER DATABASE sales_db RENAME TO business_sales;

### Explanation of the Code
- The ALTER DATABASE command is used to modify the database name.
- The RENAME TO clause specifies the new name for the database.

### When to Use
- When there is a need to change the name of a database or schema to better reflect its purpose.
- When reorganizing database structures to improve clarity and management.

### When Not to Use
- Do not use ALTER statements for renaming during peak usage times due to potential system locks.
- Avoid renaming objects that are frequently referenced in applications or scripts.

### Common Mistakes
- Forgetting to update application configurations or scripts that reference the old database or schema names.
- Not checking the impact on dependent objects such as views, procedures, or grants.

### Expected Output
**Description:** The output will be a confirmation that the database or schema has been renamed successfully. There will be no result set returned.
