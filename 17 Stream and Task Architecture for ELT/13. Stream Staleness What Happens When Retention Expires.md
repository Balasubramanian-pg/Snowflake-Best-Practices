# Stream Staleness and Retention Expiration in Snowflake

**Description:**
This topic explains what happens when a stream's retention period expires in Snowflake, and how it affects data freshness and staleness. A stream's retention period determines how long it keeps track of changes to the source table. When the retention period expires, the stream is considered stale, and its offset is reset.

## Short Explanation
**Overview:** A stream in Snowflake tracks changes to a source table for a specified retention period. When this period expires, the stream becomes stale.

**Problem Solved:** Stream staleness and retention expiration help prevent infinite storage of stream metadata, ensuring efficient use of resources.

**Importance:** Understanding stream staleness is crucial for maintaining data freshness and ensuring that analytics and data pipelines are based on recent data.

**Use Cases:**
- Real-time data integration
- Change data capture (CDC)

### Typical Scenario
A company uses a stream to capture changes to a customer database. The stream has a retention period of 24 hours. If no new changes occur within this period, the stream becomes stale, and its offset is reset.

### Core Snowflake SQL Objects Used
`CREATE STREAM`

### Code Source Execution
```sql
CREATE STREAM my_stream ON TABLE my_table FOR SYSTEM_TIME AS OF TIMESTAMP_SUB(CURRENT_TIMESTAMP, INTERVAL '1' HOUR);
```

### Example Execution Logic Step-by-Step
In this example, the stream 'my_stream' tracks changes to 'my_table' for the last hour. If no new changes occur within this hour, the stream becomes stale.

### Implementation Example
To monitor stream staleness, you can query the STREAM_PROGRESS system view:
SELECT STREAM_NAME, LAST_STALE_TIMESTAMP FROM STREAM_PROGRESS;
