# Creating Task Dependencies using the AFTER Keyword in Snowflake

**Description:**
This topic explains how to create task dependencies in Snowflake using the AFTER keyword, which allows you to schedule tasks to run after a predecessor task has completed. This feature is essential for building complex workflows and data pipelines. By using task dependencies, you can ensure that tasks are executed in a specific order, enabling efficient and reliable data processing.

### Typical Scenario
A common business scenario is creating a data pipeline that involves extracting data from multiple sources, transforming it, and loading it into a target system. By using task dependencies, you can ensure that each task in the pipeline runs after its predecessor has completed.

### Core Snowflake SQL Objects Used
`CREATE TASK`

### Code Source Execution
```sql
CREATE TASK task1
  WAREHOUSE = 'my_warehouse'
  SCHEDULE = '0 0 * * *'
AS
  CALL my_procedure();

CREATE TASK task2
  WAREHOUSE = 'my_warehouse'
  SCHEDULE = '0 0 * * *'
  AFTER task1
AS
  CALL my_procedure2();
```

### Example Execution Logic Step-by-Step
In this example, task2 is dependent on task1 and will only run after task1 has completed. The AFTER keyword is used to specify the predecessor task.
