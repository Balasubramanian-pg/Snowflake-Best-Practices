# Setting Up Cross-Account Reader Accounts for Global Cost Views

**Description:**
This topic explains how to set up cross-account reader accounts in Snowflake to enable global cost views across multiple accounts. This allows organizations to consolidate cost data from various accounts and view it in a single location.

## Short Explanation
**Overview:** Cross-account reader accounts enable organizations to share data across accounts while maintaining security and access controls.

**Problem Solved:** Organizations with multiple Snowflake accounts need to consolidate cost data for global visibility.

**Importance:** Global cost views help organizations optimize costs, allocate resources efficiently, and improve financial management.

**Use Cases:**
- Consolidating cost data from multiple departments or business units
- Enabling centralized cost monitoring and reporting across regions

### Typical Scenario
A large organization with multiple Snowflake accounts across different departments wants to view consolidated cost data in a single location to optimize resource allocation and reduce costs.

### Core Snowflake SQL Objects Used
`CREATE ACCOUNT`, `GRANT IMPORTED PRIVILEGES`

### Code Source Execution
```sql
CREATE ACCOUNT 'reader_account' FOR READER;
GRANT IMPORTED PRIVILEGES ON DATABASE 'costs' TO 'reader_account';
```

### Example Execution Logic Step-by-Step
To set up a cross-account reader account, create a new account with the 'FOR READER' clause and grant imported privileges on the database containing cost data.

### Implementation Example
Assume we have two Snowflake accounts: 'prod_account' and 'dev_account'. We want to create a cross-account reader account 'reader_account' to view cost data from both accounts.

CREATE ACCOUNT 'reader_account' FOR READER;
GRANT IMPORTED PRIVILEGES ON DATABASE 'costs' TO 'reader_account';
GRANT SELECT ON VIEW 'costs.cost_summary' TO 'reader_account';

### Explanation of the Code
- Step 1: Create a new account with the 'FOR READER' clause to designate it as a reader account.
- Step 2: Grant imported privileges on the database containing cost data to the reader account.
