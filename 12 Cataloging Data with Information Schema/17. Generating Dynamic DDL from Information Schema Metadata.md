# Generating Dynamic DDL from Information Schema Metadata in Snowflake

**Description:**
This topic explains how to dynamically generate Data Definition Language (DDL) statements in Snowflake using metadata from the Information Schema. It covers the process of extracting metadata and using it to create DDL statements programmatically.

## Short Explanation
**Overview:** Dynamic DDL generation enables automation of database object creation and modification.

**Problem Solved:** Manually writing DDL statements for multiple database objects can be time-consuming and prone to errors.

**Importance:** Dynamic DDL generation streamlines database administration and reduces the risk of human error.

**Use Cases:**
- Automating database object creation
- Generating DDL for new schema versions

### Typical Scenario
A database administrator needs to create multiple tables with similar structures in a Snowflake database. Instead of manually writing DDL statements for each table, they can use dynamic DDL generation to automate the process.

### Implementation Example
```sql
    -- Create a stored procedure to generate dynamic DDL
    CREATE OR REPLACE PROCEDURE generate_ddl(
      table_name STRING,
      column_name STRING,
      data_type STRING
    )
    RETURNS STRING
    LANGUAGE SQL
    EXECUTE AS CALLER
    BEGIN
      DECLARE ddl_stmt STRING;
      SET ddl_stmt = 'CREATE TABLE ' || table_name || ' (' || column_name || ' ' || data_type || ')';
      EXECUTE IMMEDIATE :ddl_stmt;
      RETURN 'DDL statement executed successfully';
    END;

    -- Call the stored procedure
    CALL generate_ddl('my_table', 'my_column', 'VARCHAR(100)');
  ```
