# Configuring Email Notifications for Resource Monitors in Snowflake

**Description:**
This topic explains how to set up email notifications for resource monitors in Snowflake, enabling users to receive alerts when certain thresholds are reached. Resource monitors track and alert on the usage of Snowflake resources such as credits, storage, and compute. By configuring email notifications, users can stay informed about their resource utilization and take proactive measures to manage their Snowflake environment.

## Short Explanation
**Overview:** Email notifications for resource monitors in Snowflake allow users to receive alerts when resource utilization reaches predefined thresholds.

### Typical Scenario
A Snowflake administrator wants to receive email notifications when the organization's daily credit usage exceeds $500. They create a resource monitor with an email notification threshold set to 80% of the daily credit limit.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`, `ALTER RESOURCE MONITOR`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR monitor_daily_credits
  WITH (TRIGGER_CONDITION = 80)
  NOTIFICATIONS = (EMAIL = 'admin@example.com');
```

### Example Execution Logic Step-by-Step
This SQL example creates a resource monitor called 'monitor_daily_credits' that triggers an email notification to 'admin@example.com' when daily credit usage reaches 80% of the set threshold.

### Implementation Example
To implement email notifications for resource monitors, follow these steps:
1. Create a resource monitor using the CREATE RESOURCE MONITOR statement.
2. Specify the TRIGGER_CONDITION parameter to set the threshold for email notifications.
3. Use the NOTIFICATIONS parameter to specify the email address(es) that will receive notifications.
