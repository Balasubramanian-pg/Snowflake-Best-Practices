# Defining USER_TASK_TIMEOUT_MS for Runaway Tasks in Snowflake

**Description:**
This topic explains how to prevent runaway tasks in Snowflake by defining the USER_TASK_TIMEOUT_MS variable. By setting a timeout, you can prevent tasks from consuming excessive resources and impacting system performance. This is particularly useful for long-running tasks that may not terminate properly.

## Short Explanation
**Overview:** USER_TASK_TIMEOUT_MS sets a timeout for tasks to prevent them from running indefinitely.

**Problem Solved:** Prevents runaway tasks from consuming excessive resources and impacting system performance.

**Importance:** Ensures tasks complete within a reasonable timeframe, preventing resource starvation.

- Preventing resource-intensive queries

A data engineer notices that a task is running indefinitely, causing resource contention and impacting system performance. They decide to set a timeout for the task using USER_TASK_TIMEOUT_MS to prevent similar issues in the future.
