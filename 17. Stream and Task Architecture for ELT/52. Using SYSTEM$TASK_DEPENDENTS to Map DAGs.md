# Using SYSTEM$TASK_DEPENDENTS to Map DAGs
Using SYSTEM$TASK_DEPENDENTS is a method to map Directed Acyclic Graphs (DAGs) in a database system. This system view provides information about the dependencies between tasks, which can be used to construct a DAG. By querying this view, users can gain insights into the relationships between tasks and identify potential bottlenecks or cycles in the graph. This can be particularly useful in workflow management, data processing pipelines, and other applications where tasks have complex dependencies.

## Implementation Example
```sql
SELECT * FROM SYSTEM$TASK_DEPENDENTS;
```
Alternatively, you can use the following query to get a more detailed view of the dependencies:
```sql
SELECT 
  td.task_id, 
  td.dependent_task_id, 
  t.task_name, 
  dt.task_name AS dependent_task_name
FROM 
  SYSTEM$TASK_DEPENDENTS td
  JOIN tasks t ON td.task_id = t.task_id
  JOIN tasks dt ON td.dependent_task_id = dt.task_id;
```
You can also use the following query to get the dependencies for a specific task:
```sql
SELECT * FROM SYSTEM$TASK_DEPENDENTS WHERE task_id = 'task_1';
```

## Explanation of the Code
* The `SYSTEM$TASK_DEPENDENTS` view provides information about the dependencies between tasks, including the task ID, dependent task ID, and other relevant details.
* The first query simply selects all columns from the `SYSTEM$TASK_DEPENDENTS` view, providing a basic overview of the dependencies between tasks.
* The second query joins the `SYSTEM$TASK_DEPENDENTS` view with the `tasks` table to get more detailed information about the tasks and their dependencies.
* The `JOIN` clause is used to combine rows from the `SYSTEM$TASK_DEPENDENTS` view and the `tasks` table based on the task ID.
* The `WHERE` clause is used to filter the results to only include dependencies for a specific task.
* The `SELECT` statement is used to specify the columns that should be included in the output.
* The `AS` keyword is used to assign an alias to a column, making it easier to reference in the output.

Using `SYSTEM$TASK_DEPENDENTS` to map DAGs is useful in the following scenarios:

## When not to use
