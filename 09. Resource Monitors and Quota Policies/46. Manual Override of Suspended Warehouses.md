# Manual Override of Suspended Warehouses in Snowflake

**Description:**
This topic explains how to manually override suspended warehouses in Snowflake, allowing for temporary or permanent suspension lifts to resume operations. Manual overrides are essential for managing warehouse states during maintenance or unexpected issues. They enable administrators to take control of warehouse suspension and resumption.

## Short Explanation
**Overview:** Manually overriding suspended warehouses allows administrators to regain control over warehouse operations.

**Problem Solved:** Unexpected warehouse suspension due to resource monitors or quota policies.

**Importance:** Ensures business continuity and minimizes downtime during warehouse suspension.

**Use Cases:**
- Emergency maintenance
- Unexpected resource monitor suspension

### Typical Scenario
A warehouse is automatically suspended due to a resource monitor policy. The administrator needs to manually override the suspension to resume operations.

### Core Snowflake SQL Objects Used
`ALTER WAREHOUSE`

### Code Source Execution
```sql
ALTER WAREHOUSE my_warehouse RESUME;
```

### Example Execution Logic Step-by-Step
To manually override a suspended warehouse, use the ALTER WAREHOUSE command with the RESUME option. This command allows administrators to lift the suspension and resume warehouse operations.

### Implementation Example
ALTER WAREHOUSE my_warehouse RESUME; -- Resume a suspended warehouse
ALTER WAREHOUSE my_warehouse SUSPEND; -- Suspend a warehouse

### Explanation of the Code
- Step 1: Identify the suspended warehouse using SHOW WAREHOUSES;
- Step 2: Use ALTER WAREHOUSE <warehouse_name> RESUME; to manually override the suspension.

### When to Use
- Emergency maintenance
- Unexpected resource monitor suspension
- Quota policy suspension

### When Not to Use
- Planned maintenance
- Regularly scheduled suspension

### Common Mistakes
- Forgetting to re-suspend warehouses after manual override
- Not monitoring warehouse states after manual override

### Expected Output
**Description:** The warehouse state changes to 'Running' or 'Suspended' depending on the ALTER WAREHOUSE command used.
