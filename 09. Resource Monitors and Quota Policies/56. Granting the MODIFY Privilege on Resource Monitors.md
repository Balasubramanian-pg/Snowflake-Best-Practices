# Granting the MODIFY Privilege on Resource Monitors in Snowflake

**Description:**
This topic explains how to grant the MODIFY privilege on resource monitors in Snowflake, allowing users to modify existing resource monitors. Resource monitors help track and manage compute resources and credits used by Snowflake accounts. The MODIFY privilege enables control over resource monitor configurations.

## Short Explanation
**Overview:** Granting the MODIFY privilege on resource monitors allows users to modify existing resource monitors.

**Problem Solved:** The MODIFY privilege solves the problem of controlled modifications to resource monitors, ensuring that only authorized users can make changes.

**Importance:** This privilege is important in analytics and warehousing as it helps maintain security and governance over resource utilization.

**Use Cases:**
- Granting MODIFY privilege to a data engineer
- Restricting modifications to resource monitors for security

### Typical Scenario
A Snowflake administrator wants to grant a data engineer the ability to modify an existing resource monitor without giving them full admin privileges.

`GRANT MODIFY ON RESOURCE MONITOR`

### Code Source Execution
```sql
GRANT MODIFY ON RESOURCE MONITOR my_resource_monitor TO ROLE my_data_engineer_role;
```
