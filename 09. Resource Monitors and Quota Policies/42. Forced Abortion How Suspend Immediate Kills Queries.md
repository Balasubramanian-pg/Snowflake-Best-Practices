**Description:**
This Snowflake topic explains how to immediately terminate queries using SUSPEND and KILL commands, and best practices for handling long-running or resource-intensive queries.

## Short Explanation
**Overview:** Snowflake provides mechanisms to abort queries that are consuming excessive resources or running longer than expected.

**Problem Solved:** Terminating queries that are holding up resources or taking too long to complete.

**Importance:** Prevents resource starvation and query timeouts, ensuring efficient warehouse operations.

**Use Cases:**
- Terminating a query that's consuming too much CPU
- Aborting a long-running query that's blocking other tasks

### Typical Scenario
A data analyst runs a complex query that starts consuming excessive resources, impacting other users' workloads. The admin needs to quickly terminate the query.
