# Filtering Queries by User for Departmental Chargeback

**Description:**

## Short Explanation

**Problem Solved:** It solves the problem of attributing query costs to specific users or departments, facilitating accurate chargeback and cost allocation.

**Importance:** This matters in analytics and warehousing as it helps organizations understand and manage costs associated with query usage by different departments or users.

A company wants to charge different departments for the queries executed by their employees. The company uses Snowflake to manage its data warehousing and analytics. To achieve this, they need to filter query history by user and department.

### Core Snowflake SQL Objects Used
`CREATE VIEW`, `CREATE MATERIALIZED VIEW`

### Code Source Execution
```sql
CREATE VIEW departmental_chargeback AS
SELECT 
    query_text,
    user_name,
    department,
    start_time,
    end_time,
    total_bytes
FROM 
    table(information_schema.query_history_by_session());

CREATE MATERIALIZED VIEW departmental_chargeback_mv AS
SELECT 
    department,
    SUM(total_bytes) AS total_bytes
FROM 
    departmental_chargeback
GROUP BY 
    department;
```

The example involves creating a view (departmental_chargeback) that captures query history along with the user who executed the query and their department. A materialized view (departmental_chargeback_mv) is then created to summarize the total bytes used by each department.
