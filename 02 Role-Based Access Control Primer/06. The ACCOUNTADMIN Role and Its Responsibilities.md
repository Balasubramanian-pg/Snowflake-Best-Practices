# 06. The ACCOUNTADMIN Role and Its Responsibilities

The ACCOUNTADMIN (Account Administrator) role is the highest-level role in Snowflake, encapsulating the privileges of both the SYSADMIN and SECURITYADMIN roles.
It is exclusively responsible for managing account-level parameters, viewing billing and credit consumption, and overseeing the entire Snowflake environment.

# Key Points

* It inherits all privileges from the `SYSADMIN` (object creation and management) and `SECURITYADMIN` (user, role, and grant management) roles.

* It is the only role that can view and manage account-level billing, credit usage, and account-level resource monitors by default.

* Best practices dictate that this role should not be used to create standard database objects; instead, it should be reserved strictly for account-level configuration and emergencies.

* To ensure security and prevent lockouts, at least two, but no more than five, trusted users should be granted the ACCOUNTADMIN role.

* > [!IMPORTANT] Multi-Factor Authentication (MFA) must be strictly enforced for any user assigned the ACCOUNTADMIN role to protect against unauthorized access and account compromise.

# Implementation

```sql
USE ROLE ACCOUNTADMIN;

CREATE OR REPLACE RESOURCE MONITOR account_limit_monitor
  WITH CREDIT_QUOTA = 1000
  FREQUENCY = MONTHLY
  START_TIMESTAMP = IMMEDIATELY
  TRIGGERS ON 90 PERCENT DO NOTIFY
           ON 100 PERCENT DO SUSPEND;

ALTER ACCOUNT SET RESOURCE_MONITOR = account_limit_monitor;

GRANT ROLE ACCOUNTADMIN TO USER admin_jdoe;
```

# What does the Query do?

Step 1 explanation: `USE ROLE ACCOUNTADMIN;` switches the current session context to the Account Administrator role, unlocking the highest level of account privileges required for the subsequent commands.

Step 2 explanation: `CREATE OR REPLACE RESOURCE MONITOR` provisions an account-level resource monitor to track credit consumption, notifying administrators at 90% usage and suspending all warehouses at 100%.

Step 3 explanation: `ALTER ACCOUNT SET RESOURCE_MONITOR` applies the newly created resource monitor to the entire Snowflake account, an action that can only be performed by the ACCOUNTADMIN role.

Step 4 explanation: `GRANT ROLE ACCOUNTADMIN` assigns the highest administrative privileges to a specific user (`admin_jdoe`), ensuring redundancy for account management in case another administrator is unavailable.

# When Should We Use This? 

* When configuring account-wide settings, such as network policies, account-level resource monitors, or integrations with external cloud providers.
* When reviewing billing, managing credit consumption, or performing emergency administrative overrides that lower-level roles cannot execute.
