# Manual Reclustering vs Automatic Clustering
Manual reclustering and automatic clustering are two indexing techniques used in databases to improve query performance. Manual reclustering involves reorganizing data in a table to match the order of an index, while automatic clustering is a feature that allows the database to automatically reorganize data. This can improve query performance by reducing the number of disk I/O operations required to retrieve data. Manual reclustering provides more control over the clustering process, but can be time-consuming and resource-intensive. Automatic clustering, on the other hand, is easier to implement, but may not always produce optimal results.

## Implementation Example
```sql
-- Manual Reclustering
ALTER TABLE table_name REORGANIZE;

-- Automatic Clustering
CREATE CLUSTERED INDEX index_name ON table_name (column_name);
```
Few examples and alternatives of SQL Code include using the `REORGANIZE` command to manually recluster a table, or creating a clustered index to enable automatic clustering.

## Explanation of the Code
Here are the steps involved in manual reclustering and automatic clustering:
* Manual reclustering involves reorganizing data in a table to match the order of an index, which can improve query performance.
* The `REORGANIZE` command is used to manually recluster a table, which can be time-consuming and resource-intensive.
* Automatic clustering, on the other hand, involves creating a clustered index on a table, which allows the database to automatically reorganize data.
* The `CREATE CLUSTERED INDEX` command is used to create a clustered index, which can improve query performance by reducing the number of disk I/O operations required to retrieve data.
* Clustered indexes can be created on one or more columns of a table, and can be used to improve query performance for range queries.
* The database will automatically maintain the clustered index, which can reduce the need for manual maintenance.
* However, automatic clustering may not always produce optimal results, and may require additional tuning to achieve optimal performance.

## When to use 
Here are two use cases where manual reclustering and automatic clustering can be used:
* Use manual reclustering when the database is experiencing high levels of fragmentation, and query performance is suffering as a result. Manual reclustering can help to reorganize data and improve query performance.
* Use automatic clustering when creating a new table, and you want to improve query performance without having to manually maintain the index. Automatic clustering can help to reduce the need for manual maintenance, and can improve query performance by reducing the number of disk I/O operations required to retrieve data.

## When not to use 
Here are two use cases where manual reclustering and automatic clustering should not be used:
* Do not use manual reclustering on a table that is heavily used by multiple users, as it can cause downtime and impact performance. Instead, use automatic clustering to improve query performance without impacting availability.
* Do not use automatic clustering on a table that has a high volume of insert, update, and delete operations, as it can cause the index to become fragmented and impact performance. Instead, use manual reclustering to reorganize data and improve query performance.

## Common Mistakes
Here are four common mistakes to avoid when using manual reclustering and automatic clustering:
* Failing to monitor index fragmentation, which can lead to poor query performance.
* Creating too many indexes, which can impact insert, update, and delete performance.
* Failing to maintain indexes, which can lead to poor query performance.
* Using the wrong type of index, which can impact query performance and availability.
