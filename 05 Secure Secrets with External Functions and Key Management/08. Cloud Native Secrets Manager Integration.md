# Dynamic Data Masking in Snowflake

**Description:**
Dynamic Data Masking (DDM) is a security feature in Snowflake that allows administrators to limit sensitive data exposure by masking it to unauthorized users. DDM ensures that sensitive data is not accidentally exposed to users who do not have the necessary privileges to view it. This feature helps organizations to meet compliance requirements and protect sensitive data.

## Short Explanation
**Overview:** Dynamic Data Masking is a feature that masks sensitive data at query time.

**Problem Solved:** Unauthorized access to sensitive data.

**Importance:** Protects sensitive data and meets compliance requirements.

**Use Cases:**
- Securing customer PII data\n- Protecting financial information

### Typical Scenario
A company wants to provide analysts with access to customer data, but the data contains sensitive information such as social security numbers and credit card numbers. The company uses DDM to mask this sensitive information.

`CREATE MASKING POLICY`, `ALTER TABLE`

### Code Source Execution
```sql
CREATE MASKING POLICY mask_sensitive_data AS (val string) RETURNS string -> CASE WHEN CURRENT_ROLE() = 'ANALYST' THEN val ELSE 'XXXX' END;
```

To implement DDM, first create a masking policy using the CREATE MASKING POLICY statement. Then, apply the policy to a table or column using the ALTER TABLE statement.
