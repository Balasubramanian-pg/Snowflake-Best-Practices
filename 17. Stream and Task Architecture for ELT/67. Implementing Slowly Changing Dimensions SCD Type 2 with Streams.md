# Implementing Slowly Changing Dimensions SCD Type 2 with Streams
Implementing Slowly Changing Dimensions (SCD) Type 2 with streams is a data warehousing technique used to manage changes in dimension tables over time. This approach allows for the tracking of historical changes to dimension attributes, enabling the analysis of data as it was at any point in the past. SCD Type 2 involves creating a new row in the dimension table for each change to a dimension attribute, while maintaining a history of all previous values. This approach is useful for auditing and compliance purposes, as well as for analyzing trends and patterns in data over time. By using streams to implement SCD Type 2, data can be processed in real-time, enabling up-to-the-minute analysis and decision-making.

## Implementation Example
```sql
-- Create a dimension table
CREATE TABLE dim_customer (
  customer_id INT,
  name VARCHAR(255),
  address VARCHAR(255),
  effective_date DATE,
  expiration_date DATE
);

-- Insert initial data into the dimension table
INSERT INTO dim_customer (customer_id, name, address, effective_date, expiration_date)
VALUES (1, 'John Doe', '123 Main St', '2020-01-01', '9999-12-31');

-- Create a stream to capture changes to the dimension table
CREATE STREAM customer_stream (
  customer_id INT,
  name VARCHAR(255),
  address VARCHAR(255)
);

-- Insert a change into the stream
INSERT INTO customer_stream (customer_id, name, address)
VALUES (1, 'Jane Doe', '456 Elm St');

-- Create a procedure to apply changes from the stream to the dimension table
CREATE PROCEDURE apply_changes()
BEGIN
  INSERT INTO dim_customer (customer_id, name, address, effective_date, expiration_date)
  SELECT customer_id, name, address, CURRENT_DATE, '9999-12-31'
  FROM customer_stream;

  UPDATE dim_customer
  SET expiration_date = CURRENT_DATE - 1
  WHERE customer_id IN (SELECT customer_id FROM customer_stream)
  AND expiration_date = '9999-12-31';
END;

-- Call the procedure to apply changes
CALL apply_changes();
```
Alternatively, you can use a trigger to apply changes from the stream to the dimension table automatically.

## Explanation of the Code
* The `dim_customer` table is created to store the dimension data, with columns for the customer ID, name, address, effective date, and expiration date.
* The `customer_stream` stream is created to capture changes to the dimension data, with columns for the customer ID, name, and address.
* The `apply_changes` procedure is created to apply changes from the stream to the dimension table, by inserting new rows for each change and updating the expiration date of the previous row.
* The procedure uses a `SELECT` statement to retrieve the changes from the stream, and an `INSERT` statement to add new rows to the dimension table.
* The procedure also uses an `UPDATE` statement to update the expiration date of the previous row, to indicate that it is no longer current.
* The `CALL apply_changes()` statement is used to execute the procedure and apply the changes from the stream to the dimension table.
* The use of streams and procedures allows for real-time processing and application of changes to the dimension table.

## When to use
* **Auditing and compliance**: SCD Type 2 is useful for auditing and compliance purposes, as it provides a complete history of all changes to dimension attributes.
* **Trend analysis**: SCD Type 2 is useful for analyzing trends and patterns in data over time, as it allows for the analysis of data as it was at any point in the past.
* **Real-time analytics**: The use of streams to implement SCD Type 2 enables real-time analytics and decision-making, as changes to dimension attributes can be processed and applied immediately.

## When not to use
* **Simple dimension tables**: SCD Type 2 is not necessary for simple dimension tables that do not change frequently, as it adds complexity and overhead to the data warehousing process.
* **Small datasets**: SCD Type 2 is not necessary for small datasets, as the overhead of maintaining a history of changes may not be justified by the benefits of the approach.
* **High-performance requirements**: SCD Type 2 may not be suitable for high-performance requirements, as the use of streams and procedures can add latency and overhead to the data warehousing process.

## Common Mistakes
* **Not updating the expiration date of previous rows**: Failing to update the expiration date of previous rows can result in incorrect analysis and decision-making.
* **Not handling null values**: Failing to handle null values in the stream or dimension table can result in errors and inconsistencies.
* **Not optimizing the procedure**: Failing to optimize the procedure can result in poor performance and latency.
* **Not testing the implementation**: Failing to test the implementation can result in errors and inconsistencies.
