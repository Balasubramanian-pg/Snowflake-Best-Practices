# Implementing Pay-As-You-Go Architecture with Quotas in Snowflake

**Description:**
This topic explains how to design a Pay-As-You-Go architecture in Snowflake and the importance of quotas in managing costs. A Pay-As-You-Go architecture allows users to pay only for the resources they consume. Quotas help prevent unexpected costs by limiting resource usage.

## Short Explanation
**Overview:** A Pay-As-You-Go architecture is a cloud computing pricing model where users pay only for the resources they consume.

**Problem Solved:** Unpredictable costs and resource overutilization

**Importance:** Cost-effective and efficient resource utilization

**Use Cases:**
- Data warehousing
- Data analytics
- Machine learning

### Typical Scenario
A company wants to deploy a data warehouse in Snowflake and pay only for the resources used by their analysts. They set up quotas to limit the amount of data loaded and compute resources used.

### Core Snowflake SQL Objects Used
`CREATE WAREHOUSE`, `CREATE RESOURCE MONITOR`

### Code Source Execution
```sql
CREATE WAREHOUSE my_warehouse
  WAREHOUSE_SIZE = 'XSMALL'
  AUTO_SUSPEND = 60;

CREATE RESOURCE MONITOR my_monitor
  PERCENTAGE = 80;
  TRIGGER ON 80 PERCENT;
  ACTION THROW EXCEPTION;
```

### Example Execution Logic Step-by-Step
In this example, a warehouse named 'my_warehouse' is created with a small size and auto-suspend feature. A resource monitor named 'my_monitor' is created to track usage and trigger an exception when 80% of the quota is reached.

### Implementation Example
To implement quotas in Snowflake, create a resource monitor and attach it to a warehouse. Set the quota percentage and trigger an action when the quota is reached.
