# Dynamic Data Masking in Snowflake

**Description:**
Dynamic Data Masking (DDM) is a security feature in Snowflake that allows administrators to limit sensitive data exposure by masking it to unauthorized users. DDM applies masks to data at query time, ensuring that users only see authorized data. This feature helps prevent data breaches and ensures compliance with data protection regulations.

## Short Explanation
**Overview:** Dynamic Data Masking (DDM) is a feature that masks sensitive data at query time.

**Problem Solved:** DDM solves the problem of unauthorized data access and helps prevent data breaches.

A company wants to provide analysts with access to customer data while ensuring that sensitive information such as social security numbers and credit card numbers are not exposed.

### Example Execution Logic Step-by-Step
To implement DDM, first create a masking policy using CREATE MASKING POLICY. Then, apply the policy to a table column using ALTER TABLE. When a user queries the table, Snowflake applies the mask at query time.
