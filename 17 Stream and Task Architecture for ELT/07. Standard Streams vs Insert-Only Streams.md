# Standard Streams vs Insert-Only Streams in Snowflake

**Description:**
This topic explains the difference between standard streams and insert-only streams in Snowflake. Standard streams capture all changes to a table, while insert-only streams only capture insert operations. Understanding the use cases for each stream type is crucial for effective data processing and analytics.

## Short Explanation
**Overview:** Snowflake streams allow you to capture changes to tables, but there are two types: standard and insert-only.

**Problem Solved:** Standard streams capture all changes (inserts, updates, deletes), while insert-only streams only capture inserts, reducing unnecessary processing.

**Importance:** Choosing the right stream type optimizes data processing, reduces costs, and improves analytics performance.

**Use Cases:**
- Real-time data integration
- Change data capture for analytics

### Typical Scenario
A company wants to integrate real-time data from various sources into a Snowflake data warehouse. They use standard streams to capture all changes to customer data, but use insert-only streams to capture new orders, reducing unnecessary processing.

### Core Snowflake SQL Objects Used
`CREATE STREAM`

### Code Source Execution
```sql
CREATE STREAM standard_stream ON TABLE customers;
CREATE STREAM insert_only_stream ON TABLE orders TYPE = 'INSERT_ONLY';
```

### Example Execution Logic Step-by-Step
Standard streams capture all changes: INSERT, UPDATE, DELETE. Insert-only streams only capture INSERT operations.

### Implementation Example
CREATE TABLE customers (id INT, name STRING);
CREATE TABLE orders (id INT, customer_id INT, order_date DATE);
CREATE STREAM standard_stream ON TABLE customers;
CREATE STREAM insert_only_stream ON TABLE orders TYPE = 'INSERT_ONLY';

### Explanation of the Code
- Create a standard stream on the customers table to capture all changes.
- Create an insert-only stream on the orders table to capture only insert operations.

### When to Use
- Use standard streams for comprehensive change data capture
- Use insert-only streams for high-volume insert-only tables

### When Not to Use
- Avoid using standard streams on high-volume tables with mostly inserts
- Avoid using insert-only streams when all changes need to be captured

### Common Mistakes
- Using standard streams on insert-only tables, causing unnecessary processing
- Using insert-only streams when all changes need to be captured
