# Interval Frequencies for Weekly and Monthly Quota Resets in Snowflake

**Description:**
This topic explains how to implement interval frequencies for weekly and monthly quota resets in Snowflake. It provides a guide on creating policies that automatically reset quotas at regular intervals, ensuring resource utilization and cost management. Interval frequencies help organizations enforce usage limits and optimize resource allocation.

## Short Explanation
**Overview:** Interval frequencies allow for automated quota resets at specified intervals.

**Problem Solved:** Manually tracking and resetting quotas can be inefficient and prone to errors.

**Importance:** Automated quota resets ensure fair resource utilization and help manage costs.

**Use Cases:**
- Weekly reset of data warehouse credits
- Monthly reset of compute cluster usage

### Typical Scenario
A company wants to limit its Snowflake data warehouse usage to 100 credits per week. They create a policy that resets the quota every Sunday at midnight.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`, `ALTER RESOURCE MONITOR`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR my_monitor
  WITH (CREDIT = 100)
  USING INTERVAL (1 WEEK) COMMENT = 'Weekly credit limit';

ALTER RESOURCE MONITOR my_monitor SET WEEKLY RESET;
```

### Example Execution Logic Step-by-Step
The example creates a resource monitor named 'my_monitor' with a weekly credit limit of 100 credits. The monitor is then altered to reset every week.

### Implementation Example
To implement a monthly quota reset, you can use the following SQL:

CREATE RESOURCE MONITOR my_monitor
  WITH (CREDIT = 500)
  USING INTERVAL (1 MONTH) COMMENT = 'Monthly credit limit';

ALTER RESOURCE MONITOR my_monitor SET MONTHLY RESET;

### Explanation of the Code
- Step 1: Create a resource monitor with the desired credit limit and interval.
- Step 2: Alter the resource monitor to reset at the specified interval (weekly or monthly).

### When to Use
- Enforcing weekly or monthly usage limits
- Automating quota resets for resource management

### When Not to Use
- When usage patterns are irregular or unpredictable
- For real-time quota adjustments

### Common Mistakes
- Not considering usage patterns when setting intervals
- Failing to test and validate quota resets
