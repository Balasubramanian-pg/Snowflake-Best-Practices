# Using Streams on External Tables for Data Lake Ingestion
Using streams on external tables is a powerful technique for ingesting data into a data lake. This approach allows for real-time data processing and integration, enabling organizations to make data-driven decisions quickly. By leveraging streams on external tables, data engineers can efficiently handle large volumes of data from various sources, including IoT devices, social media, and log files. This method also provides a scalable and fault-tolerant solution for data ingestion, making it an attractive option for big data analytics.

## Implementation Example
```sql
CREATE EXTERNAL TABLE external_table (
  id INT,
  name VARCHAR(255),
  timestamp TIMESTAMP
)
ROW FORMAT DELIMITED FIELDS TERMINATED BY ','
LOCATION 's3://my-bucket/data/';

CREATE STREAM stream_name (
  id INT,
  name VARCHAR(255),
  timestamp TIMESTAMP
)
FROM external_table;

INSERT INTO stream_name
SELECT id, name, timestamp
FROM external_table;
```
Few examples and alternatives of SQL Code include using different file formats such as Avro or Parquet, or using different storage locations such as HDFS or Azure Blob Storage.

## Explanation of the Code
* The `CREATE EXTERNAL TABLE` statement is used to define the schema of the external table, including the column names and data types.
* The `ROW FORMAT DELIMITED FIELDS TERMINATED BY` clause specifies the format of the data in the external table, in this case, a comma-separated values (CSV) file.
* The `LOCATION` clause specifies the storage location of the external table, in this case, an S3 bucket.
* The `CREATE STREAM` statement is used to define the stream that will be used to ingest data from the external table.
* The `FROM` clause specifies the source of the data for the stream, in this case, the external table.
* The `INSERT INTO` statement is used to insert data from the external table into the stream.
* The `SELECT` statement specifies the columns to be inserted into the stream, in this case, all columns from the external table.

## When to use
Using streams on external tables is useful in the following scenarios:
* **Real-time data processing**: When data needs to be processed in real-time, such as in IoT sensor data or social media analytics.
* **Large-scale data ingestion**: When large volumes of data need to be ingested into a data lake, such as in log file analysis or data warehousing.

## When not to use
Using streams on external tables may not be suitable in the following scenarios:
* **Small-scale data ingestion**: When small volumes of data need to be ingested, such as in a small-scale data warehousing project.
* **Batch processing**: When data can be processed in batches, such as in a nightly ETL job.

## Common Mistakes
* Not specifying the correct file format or storage location for the external table.
* Not defining the correct schema for the external table.
* Not handling errors or exceptions properly when inserting data into the stream.
* Not monitoring the stream for performance issues or data quality problems.
