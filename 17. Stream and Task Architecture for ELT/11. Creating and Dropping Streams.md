# Creating and Dropping Streams in Snowflake

**Description:**
This topic explains how to create and drop streams in Snowflake, which are used to capture changes made to a table over time.

## Short Explanation
**Overview:** Streams in Snowflake provide a change history for a table, allowing users to track changes made to the table.

**Problem Solved:** Streams solve the problem of tracking changes made to a table over time, enabling users to perform auditing, debugging, and data synchronization.

**Importance:** Streams are important in analytics and warehousing as they enable users to maintain a record of changes made to a table.

**Use Cases:**
- Auditing changes made to a table
- Data synchronization between systems
- Debugging data issues

### Typical Scenario
A business scenario where a company wants to track changes made to a customer table over time, including inserts, updates, and deletes.

### Core Snowflake SQL Objects Used
`CREATE STREAM`, `DROP STREAM`

### Code Source Execution
```sql
CREATE STREAM my_stream ON TABLE my_table;
DROP STREAM my_stream;
```

### Example Execution Logic Step-by-Step
To create a stream, use the CREATE STREAM statement, specifying the name of the stream and the table it will be based on. To drop a stream, use the DROP STREAM statement.

### Implementation Example
CREATE STREAM customer_stream ON TABLE customers;
INSERT INTO customer_stream (id, name, email) SELECT id, name, email FROM customers;
DROP STREAM customer_stream;

### Explanation of the Code
- Create a stream named customer_stream on the customers table.
- Insert data into the stream.
- Drop the stream when no longer needed.

### When to Use
- When you need to track changes made to a table over time
- When you need to perform auditing or debugging

### When Not to Use
- When you don't need to track changes made to a table
- When you're using a different method for change data capture

### Common Mistakes
- Forgetting to drop a stream when no longer needed
- Not specifying the correct table when creating a stream
