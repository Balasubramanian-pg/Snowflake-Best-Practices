# Topic

Materialized Views vs Standard Logical Views

> [!NOTE]
> Materialized views and standard logical views are two types of database objects used to simplify complex queries and improve query performance. They both provide a way to pre-compute and store the results of a query, but they differ in how they store and maintain the data.

**Short Explanation**

Materialized views and standard logical views are database objects that allow you to pre-compute and store the results of a query, improving query performance and simplifying complex queries. They differ in how they store and maintain the data, with materialized views storing the data in a physical table and standard logical views storing the data in memory.

### Example SQL Code

```sql
-- Create a materialized view
CREATE MATERIALIZED VIEW mv_example AS
SELECT column1, column2
FROM table1
WHERE condition;

-- Create a standard logical view
CREATE VIEW lv_example AS
SELECT column1, column2
FROM table1
WHERE condition;
```

### Explanation of the Code

#### Materialized View

* `CREATE MATERIALIZED VIEW`: Creates a new materialized view object.
* `mv_example`: The name of the materialized view.
* `AS`: Begins the query definition.
* `SELECT column1, column2`: Specifies the columns to be included in the materialized view.
* `FROM table1`: Specifies the source table for the materialized view.
* `WHERE condition`: Filters the rows in the source table.

#### Standard Logical View

* `CREATE VIEW`: Creates a new view object.
* `lv_example`: The name of the view.
* `AS`: Begins the query definition.
* `SELECT column1, column2`: Specifies the columns to be included in the view.
* `FROM table1`: Specifies the source table for the view.
* `WHERE condition`: Filters the rows in the source table.

#### Additional Clauses (if used)

* `WHERE`: Filters the rows in the source table.
* `GROUP BY`: Groups the rows in the source table.
* `JOIN`: Combines rows from two or more tables.
* `HAVING`: Filters the grouped rows in the source table.
* `ORDER BY`: Sorts the rows in the source table.

For each clause explain:

* `WHERE`: Filters the rows in the source table to include only those that meet the specified condition.
* `GROUP BY`: Groups the rows in the source table by one or more columns.
* `JOIN`: Combines rows from two or more tables based on a common column.
* `HAVING`: Filters the grouped rows in the source table to include only those that meet the specified condition.
* `ORDER BY`: Sorts the rows in the source table in ascending or descending order.

### When to Use

> [!TIP]
> Use materialized views when you need to:

* Pre-compute and store the results of a complex query.
* Improve query performance by reducing the amount of data that needs to be processed.
* Simplify complex queries by breaking them down into smaller, more manageable pieces.

Use standard logical views when you need to:

* Create a temporary result set that can be used in a query.
* Simplify complex queries by breaking them down into smaller, more manageable pieces.
* Create a reusable query that can be used in multiple places.

### When Not to Use

> [!WARNING]
> Avoid using materialized views when:

* The data in the source table is constantly changing and you need to reflect those changes in the materialized view.
* The materialized view is too large and takes up too much disk space.
* You need to perform complex analytics or data transformations that cannot be pre-computed.

Avoid using standard logical views when:

* You need to pre-compute and store the results of a complex query.
* You need to improve query performance by reducing the amount of data that needs to be processed.
* You need to simplify complex queries by breaking them down into smaller, more manageable pieces.

### Common Mistakes

> [!IMPORTANT]
> These errors frequently appear when using materialized views and standard logical views:

* Forgetting to refresh the materialized view when the underlying data changes.
* Not indexing the materialized view to improve query performance.
* Not using the `WHERE` clause to filter the rows in the source table.
* Not using the `GROUP BY` clause to group the rows in the source table.
* Not using the `JOIN` clause to combine rows from two or more tables.

### Summary

* Materialized views and standard logical views are two types of database objects used to simplify complex queries and improve query performance.
* Materialized views store the data in a physical table and can be used to pre-compute and store the results of a complex query.
* Standard logical views store the data in memory and can be used to create a temporary result set that can be used in a query.
* Materialized views and standard logical views can be used to simplify complex queries by breaking them down into smaller, more manageable pieces.
