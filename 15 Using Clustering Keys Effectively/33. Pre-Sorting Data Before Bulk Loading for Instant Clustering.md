# Pre-Sorting Data for Instant Clustering in Snowflake

**Description:**
Pre-sorting data before bulk loading can significantly improve query performance in Snowflake by enabling instant clustering. This involves sorting data by one or more columns before loading it into a table, allowing Snowflake to create a clustered table. By pre-sorting data, users can take advantage of Snowflake's clustering capabilities, reducing the need for additional data reorganization.

A company wants to load a large dataset into Snowflake and ensure fast query performance. They pre-sort the data by date and region, then load it into a table with a clustering key on date and region.

### Code Source Execution
```sql
CREATE TABLE sales (date DATE, region VARCHAR, sales_amount DECIMAL);
COPY INTO sales (date, region, sales_amount) FROM '@~/sales_data.csv' FILE_FORMAT = (TYPE = CSV FIELD_DELIMITER = ','),
CLUSTER BY (date, region);
```
