# Modifying Properties of a Resource Monitor with ALTER RESOURCE MONITOR

**Description:**
This topic explains how to modify the properties of an existing resource monitor in Snowflake using the ALTER RESOURCE MONITOR command. Modifying a resource monitor allows you to adjust its settings, such as the notification thresholds or the suspended and reactivated actions. This command is essential for managing and fine-tuning resource monitors to ensure efficient resource utilization.

## Short Explanation
**Overview:** The ALTER RESOURCE MONITOR command modifies the properties of an existing resource monitor.

**Problem Solved:** This command solves the problem of needing to adjust the settings of a resource monitor without having to drop and recreate it.

**Importance:** It matters in analytics or warehousing because it allows for dynamic adjustment of resource utilization and budgeting.

**Use Cases:**
- Adjusting notification thresholds for a resource monitor
- Changing the suspended and reactivated actions for a resource monitor

### Typical Scenario
A company has a resource monitor set up to track and manage the compute resources used by its data warehousing workloads. As its data processing needs change, the company needs to adjust the notification thresholds of the resource monitor to better align with its new budget and resource utilization goals.

`ALTER RESOURCE MONITOR`

### Code Source Execution
```sql
ALTER RESOURCE MONITOR my_resource_monitor SET THRESHOLD 80 PERCENT NOTIFICATION = 'suspend';
```
