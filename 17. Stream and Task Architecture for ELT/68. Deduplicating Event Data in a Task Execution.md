# Deduplicating Event Data in a Task Execution

**Description:**
Deduplicating event data in a task execution ensures that duplicate events are removed during data processing, providing a more accurate and efficient data pipeline. This process is crucial in event-driven architectures where duplicate events can cause inconsistencies and inefficiencies. By removing duplicates, data engineers can improve data quality and reduce storage costs.

### Code Source Execution
```sql

    CREATE TABLE event_data (
      event_id VARCHAR,
      event_timestamp TIMESTAMP,
      event_data VARIANT
    );

    CREATE TASK deduplicate_task
      WAREHOUSE = 'my_warehouse'
      SCHEDULE = '0 0 * * *'
      COMMENT = 'Deduplicate event data'
    AS
      INSERT INTO event_data_deduplicated
      SELECT event_id, event_timestamp, event_data
      FROM (
        SELECT event_id, event_timestamp, event_data,
               ROW_NUMBER() OVER (PARTITION BY event_id ORDER BY event_timestamp) AS row_num
        FROM event_data
      )
      WHERE row_num = 1;
  
```

### Example Execution Logic Step-by-Step
The example uses a task to deduplicate event data. The task uses a ROW_NUMBER() window function to assign a row number to each event based on the event ID and timestamp. The task then inserts only the rows with row number 1 into the deduplicated table.
