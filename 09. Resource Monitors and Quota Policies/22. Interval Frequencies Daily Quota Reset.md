# Interval Frequencies with Daily Quota Reset in Snowflake

**Description:**
This topic explains how to implement interval frequencies with a daily quota reset in Snowflake, ensuring that resource monitors and quota policies are enforced effectively.

## Short Explanation
**Overview:** Interval frequencies with daily quota reset allow for granular control over resource utilization in Snowflake.

**Problem Solved:** Prevents excessive resource usage by resetting quotas daily.

**Importance:** Crucial for managing costs and ensuring fair resource allocation.

**Use Cases:**
- Managing daily data ingestion quotas
- Enforcing daily compute resource limits

### Typical Scenario
A company wants to limit daily data ingestion from an external source to 100 GB, resetting the quota every day at midnight.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR daily_quota_monitor
  WITH INTERVAL = '1 DAY'
  QUOTA = 100
  UNIT = 'GB'
  RESET = 'USING DEFAULT RESET';
```

### Example Execution Logic Step-by-Step
This SQL statement creates a resource monitor named 'daily_quota_monitor' that tracks data ingestion in GB, with a daily quota of 100 GB. The quota resets automatically every day.

### Implementation Example
To implement this, create a resource monitor with the desired interval, quota, and unit. For example:
CREATE RESOURCE MONITOR daily_quota_monitor
  WITH INTERVAL = '1 DAY'
  QUOTA = 100
  UNIT = 'GB'
  RESET = 'USING DEFAULT RESET';
ALTER WAREHOUSE my_warehouse SET RESOURCE_MONITOR = daily_quota_monitor;

### Explanation of the Code
- Step 1: Create a resource monitor with INTERVAL = '1 DAY' to track daily usage.
- Step 2: Set QUOTA = 100 and UNIT = 'GB' to define the daily quota in GB.
- Step 3: Apply the resource monitor to a warehouse using ALTER WAREHOUSE.

### When to Use
- Daily data ingestion or processing tasks
- Enforcing daily compute resource limits

### When Not to Use
- For hourly or minute-level quota resets
- For quotas that don't reset daily

### Common Mistakes
- Not specifying the RESET option
- Using incorrect units for quota
