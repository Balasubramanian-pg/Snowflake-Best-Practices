# Identifying Non-Compliant Object Names via Information Schema in Snowflake

**Description:**
This topic explains how to identify object names in Snowflake that do not comply with a specified naming convention using the Information Schema. It provides a SQL query example to detect such non-compliant object names.

## Short Explanation
**Overview:** The Information Schema in Snowflake provides a way to query metadata about database objects.

**Problem Solved:** Identifying object names that do not follow a specific naming convention.

**Importance:** Ensures data consistency and adherence to organizational standards.

### Typical Scenario
A Snowflake administrator wants to identify all tables in a schema whose names do not start with a specific prefix, such as 'tbl_' or 'dim_'

### Core Snowflake SQL Objects Used
`INFORMATION_SCHEMA.TABLES`

### Code Source Execution
```sql
SELECT table_name FROM INFORMATION_SCHEMA.TABLES WHERE table_type = 'BASE TABLE' AND table_name NOT LIKE 'tbl_%';
```

### Example Execution Logic Step-by-Step
The query selects table names from the INFORMATION_SCHEMA.TABLES view where the table type is 'BASE TABLE' and the table name does not match the specified pattern 'tbl_%'.

### Implementation Example
To identify non-compliant table names, execute the following SQL query:
```
SELECT table_name FROM INFORMATION_SCHEMA.TABLES WHERE table_type = 'BASE TABLE' AND table_name NOT LIKE 'tbl_%';
```

### Explanation of the Code
- The INFORMATION_SCHEMA.TABLES view contains metadata about tables in the current database.
- The table_type = 'BASE TABLE' filter ensures that only base tables are considered, excluding views and other object types.
- The table_name NOT LIKE 'tbl_%' filter identifies table names that do not start with 'tbl_'.

### When to Use
- During data governance audits
- When enforcing data naming conventions

### When Not to Use
- When the naming convention is not critical
- In development environments where object names are frequently changed
