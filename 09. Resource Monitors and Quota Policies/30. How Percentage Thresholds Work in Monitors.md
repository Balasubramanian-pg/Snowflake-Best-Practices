# Understanding Percentage Thresholds in Snowflake Monitors

**Description:**
This topic explains how percentage thresholds work in Snowflake monitors, which allow you to set alerts based on relative changes in metrics. By using percentage thresholds, you can define more flexible and dynamic alerting conditions. This feature is useful for monitoring and reacting to changes in your data over time.

## Short Explanation
**Overview:** Percentage thresholds in Snowflake monitors enable you to set alert conditions based on percentage changes in metrics.

**Problem Solved:** Traditional threshold-based alerts can be inflexible and may not account for natural fluctuations in data. Percentage thresholds solve this problem by allowing you to define alert conditions relative to the current value of a metric.

**Importance:** This feature is important in analytics and warehousing because it enables more sophisticated and adaptive monitoring and alerting.

**Use Cases:**
- Detecting anomalies in data usage patterns
- Monitoring changes in data volume or query performance

### Typical Scenario
A business wants to monitor its daily data usage and receive alerts when usage increases or decreases by more than 20% compared to the previous day. Using Snowflake monitors with percentage thresholds, the business can set up alerts that automatically adjust to changing data patterns.
