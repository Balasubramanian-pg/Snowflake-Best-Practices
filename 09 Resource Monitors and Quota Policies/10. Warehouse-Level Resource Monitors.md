# Warehouse-Level Resource Monitors in Snowflake

**Description:**
This topic explains how to configure and utilize warehouse-level resource monitors in Snowflake to track and manage compute resources, specifically for warehouses. Resource monitors help in optimizing costs and performance by setting limits on credits used.

## Short Explanation
**Overview:** Warehouse-level resource monitors allow you to set credit quotas for individual warehouses, alerting you when thresholds are reached.

**Problem Solved:** Managing and controlling compute costs associated with warehouses.

**Importance:** Crucial for budgeting, cost control, and performance optimization in Snowflake.

**Use Cases:**
- Controlling costs for compute-intensive ETL processes
- Monitoring and alerting for unusual warehouse usage

### Typical Scenario
A company sets up a resource monitor on a warehouse used for daily data processing to ensure it does not exceed a certain credit threshold, alerting the team to take action if usage spikes.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`, `ALTER WAREHOUSE`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR my_monitor FOR WAREHOUSE my_warehouse WITH CREDIT QUOTA = 100;
```

### Example Execution Logic Step-by-Step
To create a resource monitor for a warehouse, you first create the monitor with a specified credit quota, then apply it to a warehouse. For example:

```sql
CREATE RESOURCE MONITOR my_monitor FOR WAREHOUSE my_warehouse WITH CREDIT QUOTA = 100;
ALTER WAREHOUSE my_warehouse SET RESOURCE MONITOR = my_monitor;
```

### Implementation Example
To fully implement a warehouse-level resource monitor:
1. Create a resource monitor with a specified credit quota.
2. Apply the monitor to a warehouse.
3. Configure alerts as needed.

```sql
CREATE RESOURCE MONITOR finance_monitor FOR WAREHOUSE finance_wh WITH CREDIT QUOTA = 50;
ALTER WAREHOUSE finance_wh SET RESOURCE MONITOR = finance_monitor;
```

### Explanation of the Code
- Step 1: Create a resource monitor named 'finance_monitor' for warehouse 'finance_wh' with a credit quota of 50.
- Step 2: Apply the 'finance_monitor' resource monitor to the 'finance_wh' warehouse.

### When to Use
- When you need to manage costs for specific warehouses
- To monitor and alert on warehouse credit usage

### When Not to Use
- For account-level monitoring; use account monitors instead
- If you want to monitor storage rather than compute resources

### Common Mistakes
- Setting too low of a credit quota, causing warehouses to suspend
- Not setting up alerts, leading to unexpected costs
