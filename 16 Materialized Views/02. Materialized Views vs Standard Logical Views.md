# Materialized Views vs Standard Logical Views

> [!NOTE]
> Materialized views and standard logical views are two types of database objects that serve different purposes. Materialized views are precomputed results of a query, stored in a physical table, while standard logical views are virtual tables that represent the result of a query without storing the actual data.

**Short Explanation**

Materialized views and standard logical views are both used to simplify complex queries and improve database performance, but they differ in how they store and retrieve data. Materialized views store the result of a query in a physical table, while standard logical views do not store the data and instead retrieve it on demand.

### Example SQL Code

```sql
-- Materialized View
CREATE MATERIALIZED VIEW mv_example AS
SELECT column_name
FROM table_name
WHERE condition;

-- Standard Logical View
CREATE VIEW lv_example AS
SELECT column_name
FROM table_name
WHERE condition;
```

### Explanation of the Code

#### Materialized View

* The `CREATE MATERIALIZED VIEW` statement creates a new materialized view named `mv_example`.
* The `AS` keyword specifies the query that will be executed to populate the materialized view.
* The `SELECT` statement retrieves the specified column from the dataset.
* The `FROM` clause defines the source table that contains the data.
* The `WHERE` clause filters rows so that only records meeting the condition appear in the result.

#### Standard Logical View

* The `CREATE VIEW` statement creates a new standard logical view named `lv_example`.
* The `AS` keyword specifies the query that will be executed to populate the logical view.
* The `SELECT` statement retrieves the specified column from the dataset.
* The `FROM` clause defines the source table that contains the data.
* The `WHERE` clause filters rows so that only records meeting the condition appear in the result.

#### Additional Clauses (if used)

* `WHERE` clause: filters rows based on a condition.
* `GROUP BY` clause: groups rows based on one or more columns.
* `JOIN` clause: combines rows from two or more tables based on a common column.
* `HAVING` clause: filters groups based on a condition.
* `ORDER BY` clause: sorts rows based on one or more columns.

### When to Use

> [!TIP]
> Use materialized views when you need to:

* Improve query performance by reducing the number of joins and subqueries.
* Simplify complex queries by breaking them down into smaller, more manageable pieces.
* Create a physical table that can be used as a data source for other queries.

Use standard logical views when you need to:

* Create a virtual table that represents the result of a query without storing the actual data.
* Simplify complex queries by breaking them down into smaller, more manageable pieces.
* Create a temporary table that can be used for ad-hoc queries.

### When Not to Use

> [!WARNING]
> Avoid using materialized views in the following situations:

* When the underlying data changes frequently, as materialized views can become outdated.
* When the query is complex and requires frequent updates, as materialized views can become cumbersome to maintain.
* When the data is too large to fit in memory, as materialized views require physical storage.

Avoid using standard logical views in the following situations:

* When row-level detail must remain unchanged, as standard logical views can introduce data inconsistencies.
* When a different SQL feature performs the task more efficiently, such as using a stored procedure or a function.
* When the transformation should occur earlier in the data pipeline, as standard logical views can introduce unnecessary complexity.

### Common Mistakes

> [!IMPORTANT]
> These errors frequently appear when using materialized views and standard logical views:

* Forgetting to update the materialized view when the underlying data changes.
* Referencing columns that do not exist in the materialized view or standard logical view.
* Mixing aggregate and non-aggregate columns incorrectly in the materialized view or standard logical view.
* Confusing similar SQL constructs, such as using a materialized view instead of a standard logical view.

### Summary

* Materialized views store the result of a query in a physical table, while standard logical views do not store the data and instead retrieve it on demand.
* Use materialized views to improve query performance and simplify complex queries, and use standard logical views to create virtual tables and simplify complex queries.
* Avoid using materialized views when the underlying data changes frequently, and avoid using standard logical views when row-level detail must remain unchanged.
