# Scheduling Tasks using Interval Syntax in Snowflake

**Description:**
This topic explains how to schedule tasks in Snowflake using interval syntax, allowing for automated execution of SQL queries at regular intervals.

## Short Explanation
**Overview:** Snowflake's task scheduling feature enables users to automate SQL query execution at specified intervals.

**Problem Solved:** Manually running SQL queries at regular intervals can be time-consuming and prone to errors.

**Importance:** Automating tasks using interval syntax improves efficiency, reduces manual errors, and enables real-time data processing.

**Use Cases:**
- Scheduling daily data loads
- Running periodic data aggregations

### Typical Scenario
A business needs to load new sales data into their Snowflake warehouse every hour to ensure up-to-date analytics and reporting.

### Core Snowflake SQL Objects Used
`CREATE TASK`

### Code Source Execution
```sql
CREATE TASK load_sales_data
  WAREHOUSE = 'my_warehouse'
  SCHEDULE = '0 1 * * * *'  // every hour
  COMMENT = 'Load new sales data'
  CALL load_sales_data_sp();
```

### Example Execution Logic Step-by-Step
This example creates a task named 'load_sales_data' that runs every hour using the specified schedule. The task calls a stored procedure 'load_sales_data_sp' to load new sales data.

CREATE TABLE sales_data (id INT, date DATE);
CREATE STORED PROCEDURE load_sales_data_sp()
  RETURNS INT
  LANGUAGE JAVASCRIPT
  EXECUTE AS CALLER
  RETURNS NULL ON NULL INPUT
BEGIN
  // load sales data logic
  return 1;
END;
CREATE TASK load_sales_data
  WAREHOUSE = 'my_warehouse'
  SCHEDULE = '0 1 * * * *'  // every hour
  COMMENT = 'Load new sales data'
  CALL load_sales_data_sp();
