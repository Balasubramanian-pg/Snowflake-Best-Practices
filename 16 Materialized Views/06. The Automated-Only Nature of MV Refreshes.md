# The Automated Only Nature of MV Refreshes

> [!NOTE]
> MV Refreshes are a type of data refresh mechanism used in data warehousing and business intelligence to update materialized views (MV) in a database. This feature automates the process of refreshing MVs, ensuring that the data remains up-to-date and accurate.

**Short Explanation**

MV Refreshes solve the problem of manual data refreshes, which can be time-consuming and prone to errors. This feature is important in databases and analytics as it enables automated data updates, reducing the risk of data inconsistencies and improving the overall data quality. MV Refreshes are commonly used in real-world workflows, such as data warehousing, business intelligence, and data analytics.

### Example SQL Code

```sql
CREATE MATERIALIZED VIEW mv_name
REFRESH COMPLETE ON DEMAND
AS
SELECT column_name
FROM table_name;
```

### Explanation of the Code

#### CREATE MATERIALIZED VIEW

* Creates a new materialized view named `mv_name`.
* The `REFRESH COMPLETE ON DEMAND` clause specifies that the MV should be refreshed automatically when the underlying data changes.

#### SELECT

* Retrieves the specified column from the dataset.
* The `AS` keyword is used to define the query that will be executed to populate the MV.

#### REFRESH COMPLETE ON DEMAND

* Specifies that the MV should be refreshed automatically when the underlying data changes.
* The `ON DEMAND` clause means that the refresh will occur when the MV is queried.

### When to Use

> [!TIP]
> Use MV Refreshes when you need to automate the process of refreshing materialized views in a database.

Typical use cases:

* Creating automated data refreshes for data warehouses and business intelligence systems
* Updating materialized views in real-time to reflect changes in the underlying data
* Improving data quality and consistency by reducing manual data refreshes

Example scenarios:

* Creating a data warehouse that automatically refreshes its materialized views every hour
* Building a business intelligence system that updates its materialized views in real-time to reflect changes in sales data
* Improving data quality by automating the process of refreshing materialized views in a data analytics system

### When Not to Use

> [!WARNING]
> Avoid using MV Refreshes in situations where manual data refreshes are necessary or where the refresh process should occur at a specific time.

Examples:

* When row-level detail must remain unchanged and manual refreshes are necessary to ensure data integrity
* When a different data refresh mechanism, such as a scheduled job, performs the task more efficiently
* When the transformation should occur earlier in the data pipeline and manual refreshes are necessary to ensure data accuracy

### Common Mistakes

> [!IMPORTANT]
> These errors frequently appear when using MV Refreshes.

Examples:

* Forgetting to specify the `REFRESH COMPLETE ON DEMAND` clause, resulting in manual refreshes
* Referencing columns that do not exist in the underlying table, resulting in errors during the refresh process
* Mixing aggregate and non-aggregate columns incorrectly, resulting in incorrect data in the MV
* Confusing similar SQL constructs, such as `REFRESH COMPLETE ON DEMAND` and `REFRESH FAST ON DEMAND`
