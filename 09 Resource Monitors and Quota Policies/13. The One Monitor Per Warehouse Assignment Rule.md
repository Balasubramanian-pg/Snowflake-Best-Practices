# The One Monitor Per Warehouse Assignment Rule in Snowflake

**Description:**
The One Monitor Per Warehouse Assignment Rule is a best practice in Snowflake that ensures each warehouse is monitored by only one resource monitor. This rule helps prevent multiple monitors from being attached to a single warehouse, which can lead to confusion and inconsistent monitoring.

## Short Explanation
**Overview:** The One Monitor Per Warehouse Assignment Rule ensures a single point of monitoring for each warehouse.

**Problem Solved:** Prevents multiple resource monitors from being attached to a single warehouse.

**Importance:** Ensures clear and consistent monitoring of warehouses.

**Use Cases:**
- Assigning a single resource monitor to a development warehouse
- Applying a single resource monitor to a production warehouse

### Typical Scenario
A company has multiple warehouses for different environments (e.g., development, testing, production). They want to ensure each warehouse is monitored by a single resource monitor to track its usage and costs effectively.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`, `ALTER WAREHOUSE`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR monitor_dev_warehouse
  CREDENTIALS = (AWS_S3_BUCKET = 'my-bucket' AWS_S3_KEY_ID = 'my-key-id' AWS_S3_SECRET_KEY = 'my-secret-key')
  COMMENT = 'Monitor for dev warehouse';

ALTER WAREHOUSE dev_warehouse
  SET RESOURCE_MONITOR = monitor_dev_warehouse;
```
