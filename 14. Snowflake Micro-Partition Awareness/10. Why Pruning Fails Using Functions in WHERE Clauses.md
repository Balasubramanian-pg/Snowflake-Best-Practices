# Why Pruning Fails When Using Functions in WHERE Clauses

**Description:**
This topic explains how using functions in WHERE clauses can prevent effective pruning in Snowflake, leading to performance issues. It discusses the implications of using functions in filtering conditions and provides best practices for optimizing queries.

## Short Explanation
**Overview:** Using functions in WHERE clauses can prevent Snowflake from pruning files during query execution.

**Problem Solved:** Understanding why pruning fails with functions in WHERE clauses helps optimize queries for better performance.

**Importance:** This matters in analytics and warehousing because efficient queries are crucial for fast data insights.

**Use Cases:**
- Filtering data based on calculated values
- Using functions to transform data for comparison

### Typical Scenario
A business scenario where a query is executed frequently, but its performance is suboptimal due to the use of a function in the WHERE clause, leading to slower-than-expected insights from data.

### Core Snowflake SQL Objects Used
`CREATE TABLE`, `SELECT`

### Code Source Execution
```sql
SELECT * FROM mytable WHERE upper(name) = 'JOHN'
```

### Example Execution Logic Step-by-Step
When a function like upper() is used in a WHERE clause, Snowflake cannot prune files because it cannot determine which files contain the required data without scanning all files.

### Implementation Example
Instead of using functions in WHERE clauses, consider transforming data during ingestion or creating materialized views to pre-compute values.

### Explanation of the Code
- The query attempts to filter rows where the name is 'JOHN' (case-insensitive).
- However, because the upper() function is used, Snowflake must scan all files to determine which rows match the condition.

### When to Use
- When data transformation is necessary for filtering and performance is not critical

### When Not to Use
- For frequently executed queries where performance is critical
- When dealing with large datasets where pruning is essential for performance

### Common Mistakes
- Using functions in WHERE clauses without considering performance implications
- Not testing query performance with different filtering conditions
