# Metadata-Only Queries and Zero-Compute Micro-Partition Scans in Snowflake

**Description:**
Metadata-only queries in Snowflake enable fast, zero-compute scans of micro-partitions to retrieve metadata without scanning the actual data. This feature is useful for quickly gathering information about the structure and organization of data in Snowflake. It helps in optimizing storage and performance by understanding how data is micro-partitioned.

## Short Explanation
**Overview:** Metadata-only queries allow users to scan micro-partitions without loading data into memory, which can significantly speed up certain types of queries.

**Problem Solved:** The problem solved by metadata-only queries is the need to quickly retrieve information about data organization and structure without incurring the cost of loading the actual data.

**Importance:** This feature is important in analytics and warehousing as it enables efficient data management and optimization.

**Use Cases:**
- Quickly determining the micro-partitioning scheme of a table
- Gathering statistics about data distribution without scanning the data
