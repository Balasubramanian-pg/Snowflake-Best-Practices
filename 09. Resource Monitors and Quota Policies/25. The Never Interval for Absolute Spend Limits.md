# The Never Interval for Absolute Spend Limits in Snowflake

**Description:**
The 'Never Interval' concept in Snowflake's Resource Monitors allows for setting absolute spend limits with flexible time intervals. This feature helps prevent unexpected costs by enforcing spending thresholds. The Never Interval specifically implies that no time interval is set, allowing for immediate or flexible limit enforcement.

## Short Explanation
**Overview:** The Never Interval in Snowflake's Resource Monitors allows users to set absolute spend limits without a specific time interval.

**Problem Solved:** Prevents unexpected costs by enforcing spending thresholds without requiring a fixed time interval.

**Importance:** Essential for budgeting and cost control in Snowflake, especially for unpredictable or variable workloads.

**Use Cases:**
- Setting a maximum spend limit for a specific role or user without a recurring interval
- Enforcing an immediate spend limit on a resource monitor

### Typical Scenario
A company wants to ensure that a specific role in Snowflake never exceeds a certain spend limit, regardless of the time interval. They create a resource monitor with an absolute spend limit and set the interval to 'Never'.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR monitor_name WITH PARAMETERS
  CREDIT_QUOTA = 100,
  INTERVAL = 'NEVER',
  THRESHOLD = 80;
ALTER ACCOUNT SET RESOURCE_MONITOR = monitor_name;
```

The provided SQL example creates a resource monitor named 'monitor_name' with a credit quota of 100, an interval of 'NEVER', and a threshold of 80%. The ACCOUNT is then altered to set this resource monitor.

### Implementation Example
To implement this, a Snowflake admin would execute the SQL command above, replacing 'monitor_name' with a descriptive name. They would then monitor the resource usage to ensure it stays within the set limits.

### Explanation of the Code
- The 'CREATE RESOURCE MONITOR' statement establishes a new monitor.
- The 'CREDIT_QUOTA' parameter sets the absolute spend limit.
- The 'INTERVAL' set to 'NEVER' implies the limit is enforced without a recurring interval.
- The 'THRESHOLD' parameter defines the percentage at which an alert is triggered.

### When to Use
- When you need to enforce a strict, absolute spend limit without a specific recurring interval.
- For roles or users with unpredictable spending patterns.

### When Not to Use
- When a recurring interval for spend limits makes more sense, such as monthly or weekly limits.
- For resources with predictable and consistent usage patterns.

### Common Mistakes

- Not setting up alerts properly when the threshold is reached.
