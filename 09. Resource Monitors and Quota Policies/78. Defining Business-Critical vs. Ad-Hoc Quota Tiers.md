# Defining Business-Critical vs. Ad-Hoc Quota Tiers in Snowflake

**Description:**
This topic explains how to categorize and manage quota tiers in Snowflake, distinguishing between business-critical and ad-hoc quotas to ensure efficient resource allocation and cost management.

## Short Explanation
**Overview:** Snowflake allows administrators to set quota policies to manage resource utilization and costs.

**Problem Solved:** Without proper quota tier management, resources can be misallocated, leading to unexpected costs and performance issues.

**Importance:** Effective quota tier management ensures that business-critical workloads receive sufficient resources while preventing ad-hoc queries from consuming excessive resources.

**Use Cases:**
- Data warehousing for enterprise analytics
- Managing resource-intensive data science workloads

### Typical Scenario
A company has a Snowflake account with multiple teams using the platform for various workloads. The company wants to ensure that business-critical analytics workloads receive sufficient resources while preventing ad-hoc queries from overwhelming the system.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`, `ALTER WAREHOUSE SET RESOURCE_MONITOR`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR my_business_critical_monitor
  WITH CREDIT_QUOTA = 100
  TRIGGER ON 80 PERCENT
  ACTION THROW_EXCEPTION;

ALTER WAREHOUSE my_warehouse SET RESOURCE_MONITOR = my_business_critical_monitor;
```

### Example Execution Logic Step-by-Step
This example creates a resource monitor called `my_business_critical_monitor` with a credit quota of 100. When the monitor reaches 80% of its quota, it throws an exception. The `my_warehouse` is then altered to use this resource monitor.

### Implementation Example
To implement business-critical vs. ad-hoc quota tiers, create separate resource monitors with different credit quotas and triggers. Assign these monitors to warehouses based on the type of workload.

### Explanation of the Code
- Step 1: Create a resource monitor with a specific credit quota.
- Step 2: Alter a warehouse to use the resource monitor.

### When to Use
- When managing multiple teams with different resource requirements
- When ensuring business-critical workloads receive sufficient resources

### When Not to Use
- When there are no resource constraints or budget limitations
- When all workloads have similar resource requirements

### Common Mistakes
- Not setting realistic credit quotas for business-critical workloads
- Failing to monitor and adjust resource monitors regularly
