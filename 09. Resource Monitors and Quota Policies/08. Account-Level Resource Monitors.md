# Account-Level Resource Monitors in Snowflake

**Description:**
Account-Level Resource Monitors allow you to track and manage resource usage across your entire Snowflake account. These monitors provide a centralized way to monitor and control resource utilization, enabling you to optimize your Snowflake environment and prevent resource starvation.

## Short Explanation
**Overview:** Account-Level Resource Monitors track resource usage across an entire Snowflake account.

**Problem Solved:** Resource starvation and lack of visibility into account-wide resource utilization.

**Importance:** Ensures optimal resource allocation, prevents over-usage, and enables proactive management.

**Use Cases:**
- Monitoring and managing compute resources
- Tracking storage usage
- Setting quotas for departments or teams

### Typical Scenario
A Snowflake administrator wants to monitor and manage resource usage across multiple roles, warehouses, and databases in their account. They create an account-level resource monitor to track compute resources and storage usage, and set thresholds to alert them when usage exceeds predetermined limits.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR monitor_compute
  WITH (COMMENT = 'Monitor compute resources')
  USING (WAREHOUSE = 'LOAD_WH');
```

### Example Execution Logic Step-by-Step
The example creates an account-level resource monitor named 'monitor_compute' that tracks compute resources used by the 'LOAD_WH' warehouse.

### Implementation Example
CREATE RESOURCE MONITOR monitor_storage
  WITH (COMMENT = 'Monitor storage usage')
  USING (STORAGE);

ALTER ACCOUNT SET RESOURCE_MONITOR = monitor_storage;

### Explanation of the Code
- Create a resource monitor named 'monitor_storage' to track storage usage.
- Apply the resource monitor to the account using the ALTER ACCOUNT statement.

### When to Use
- When you need to track resource usage across multiple roles and warehouses
- When you want to set quotas for departments or teams

### When Not to Use
- When you only need to monitor resource usage for a single role or warehouse
- When you have a small Snowflake environment with minimal resource usage

- Not regularly reviewing and adjusting resource monitor settings
