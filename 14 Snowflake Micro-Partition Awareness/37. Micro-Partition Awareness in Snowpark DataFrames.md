# Micro-Partition Awareness in Snowpark DataFrames

**Description:**
Micro-partition awareness in Snowpark DataFrames enables efficient data processing by leveraging Snowflake's columnar storage and micro-partitioning. This feature allows Snowpark to prune micro-partitions, reducing the amount of data that needs to be processed. By doing so, it improves query performance and reduces costs.

A data analyst wants to filter a large dataset stored in a Snowflake table that is micro-partitioned by date. ### Implementation Example

import snowflake.snowpark.functions as F
from snowflake.snowpark import Session

session = Session.builder.appName('My App').build()
df = session.sql('SELECT * FROM mytable')

df.show()
