# Defining a Clustering Key on Table Creation in Snowflake

**Description:**
This topic explains how to define a clustering key when creating a table in Snowflake, which helps optimize data storage and query performance. A clustering key determines the order in which data is stored in a table, allowing for more efficient data retrieval. By defining a clustering key during table creation, you can improve query performance and reduce storage costs. This is a crucial aspect of effective data management in Snowflake.

## Short Explanation
**Overview:** Defining a clustering key on table creation allows you to control the physical storage of data in Snowflake.

**Problem Solved:** Without a clustering key, Snowflake stores data in a random order, which can lead to slower query performance.

**Importance:** Proper clustering key definition is essential for optimizing query performance, reducing storage costs, and improving data management in Snowflake.

### Typical Scenario
A company wants to store customer data in a Snowflake table, with queries frequently filtering on the 'country' and 'region' columns. By defining a clustering key on these columns during table creation, the company can improve query performance and reduce storage costs.

### Example Execution Logic Step-by-Step
In this example, the 'customers' table is created with a clustering key defined on the 'country' and 'region' columns. This tells Snowflake to store data in a way that groups rows with the same 'country' and 'region' values together.
