To demonstrate the concept, consider the following example: a user with the EXECUTE TASK privilege can execute a task using the ALTER TASK command with the EXECUTE option.

### Implementation Example
CREATE ROLE data_engineer;
GRANT EXECUTE TASK ON ACCOUNT TO ROLE data_engineer;
CREATE TASK daily_data_load
  WAREHOUSE = 'my_warehouse'
  SCHEDULE = '0 8 * * *'
  COMMENT = 'Daily data load task'
  AS
  COPY INTO my_target_table
  FROM my_source_table;
GRANT EXECUTE ON TASK daily_data_load TO ROLE data_engineer;

### Explanation of the Code
- Create a new role for the data engineer
- Grant the EXECUTE TASK privilege to the data engineer role
- Create a new task for daily data loading
- Grant execute permission on the task to the data engineer role
