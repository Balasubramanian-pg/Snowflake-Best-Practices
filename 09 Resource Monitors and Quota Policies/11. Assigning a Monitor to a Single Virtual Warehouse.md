# Assigning a Monitor to a Single Virtual Warehouse in Snowflake

**Description:**
This topic explains how to assign a resource monitor to a single virtual warehouse in Snowflake, allowing for more granular control over warehouse usage and costs. By assigning a monitor to a specific virtual warehouse, you can track and manage its usage more effectively. This is particularly useful in environments with multiple virtual warehouses with different usage patterns.

## Short Explanation
**Overview:** Assigning a resource monitor to a single virtual warehouse enables detailed usage tracking and cost management.

**Problem Solved:** Inability to track and manage usage of individual virtual warehouses, leading to potential cost and performance issues.

**Importance:** Effective resource utilization and cost management are critical in Snowflake environments with multiple virtual warehouses.

**Use Cases:**
- Assigning monitors to development and production virtual warehouses separately
- Tracking usage of virtual warehouses with different usage patterns

### Typical Scenario
A company has multiple virtual warehouses for different business units. They want to track and manage the usage of each virtual warehouse separately to optimize costs and performance.

### Core Snowflake SQL Objects Used
`CREATE RESOURCE MONITOR`, `ALTER WAREHOUSE`

### Code Source Execution
```sql
CREATE RESOURCE MONITOR my_monitor
  WAREHOUSES = (my_warehouse)
  TRIGGERS = (30 * 60 = 80, 60 * 60 = 100);

ALTER WAREHOUSE my_warehouse SET RESOURCE_MONITOR = my_monitor;
```

### Example Execution Logic Step-by-Step
To assign a monitor to a single virtual warehouse, first create a resource monitor using the CREATE RESOURCE MONITOR statement, specifying the warehouse and triggers. Then, use the ALTER WAREHOUSE statement to set the resource monitor for the specific virtual warehouse.
