# Attributing Time Travel Costs via Storage Views

**Description:**
This topic explains how to attribute costs associated with Time Travel in Snowflake using storage views. It provides a method to track and allocate costs related to data retention and versioning. By utilizing storage views, users can efficiently manage and monitor their Time Travel costs.

A company wants to track the costs associated with storing historical versions of their data. They use Snowflake's Time Travel feature and want to attribute these costs to specific departments or projects.

### Core Snowflake SQL Objects Used
`CREATE VIEW`, `SHOW STORAGE`

### Code Source Execution
```sql

    CREATE VIEW time_travel_costs AS
    SELECT 
      t.name AS table_name,
      t.schema_name,
      SUM(s.storage_bytes) AS total_storage_bytes,
      SUM(s.storage_bytes) * 0.000022 / 720 AS estimated_cost
    FROM 
      TABLE(INFORMATION_SCHEMA.TABLES) t
    JOIN 
      SHOW STORAGE FOR TABLE t.name IN SCHEMA t.schema_name
    AS s ON s.name = t.name
    GROUP BY 
      t.name, t.schema_name
  
```

### Example Execution Logic Step-by-Step
The provided SQL creates a view that joins the INFORMATION_SCHEMA.TABLES with the SHOW STORAGE output. It calculates the total storage used by each table and estimates the cost based on the storage bytes.
