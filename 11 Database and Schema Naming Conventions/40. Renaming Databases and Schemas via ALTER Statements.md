# Renaming Databases and Schemas in Snowflake Using ALTER Statements

**Description:**
This topic explains how to rename databases and schemas in Snowflake using ALTER statements. Renaming these objects can help with organization, clarity, and adherence to naming conventions. Snowflake allows renaming databases and schemas using specific ALTER statements.

## Short Explanation
**Overview:** Renaming databases and schemas in Snowflake is achieved through ALTER statements.

**Problem Solved:** The ability to rename databases and schemas helps in reorganizing and standardizing the structure of Snowflake accounts.

**Importance:** Proper naming conventions enhance visibility, manageability, and understanding of the data structure within Snowflake.

**Use Cases:**
- Reorganizing database and schema names for clarity
- Adhering to new company-wide naming conventions

### Typical Scenario
A company decides to rename one of its databases from 'old_database' to 'new_database' and a schema from 'public' to 'data_warehouse' to better reflect their current data structure and usage.

### Core Snowflake SQL Objects Used
`ALTER DATABASE`, `ALTER SCHEMA`

### Code Source Execution
```sql
ALTER DATABASE old_database RENAME TO new_database;
ALTER SCHEMA my_schema RENAME TO data_warehouse;
```

### Example Execution Logic Step-by-Step
To rename a database or schema in Snowflake, you use the ALTER statement followed by the object type (DATABASE or SCHEMA), the current name of the object, and the RENAME TO clause with the new name. For example, to rename a database named 'old_database' to 'new_database', you would execute: ALTER DATABASE old_database RENAME TO new_database;

### Implementation Example
Suppose we have a database named 'sales_db' and we want to rename it to 'customer_data'. Similarly, we have a schema named 'public' under 'sales_db' and we want to rename it to 'sales_schema'.

ALTER DATABASE sales_db RENAME TO customer_data;
ALTER SCHEMA customer_data.public RENAME TO sales_schema;

### Explanation of the Code
- The first step is to identify the current names of the database and schema you want to rename.
- The second step is to construct the ALTER statement with the RENAME TO clause for each object.

### When to Use
- When there is a need to reorganize or standardize database and schema names
- When renaming databases and schemas to reflect changes in data structure or organization

### When Not to Use
- When changing names could disrupt dependent objects or user permissions
- When frequent renaming could cause confusion

### Common Mistakes
- Forgetting to update dependent objects' references to the renamed database or schema
- Not considering the impact on data lineage and documentation
