### Example Execution Logic Step-by-Step
Micro-partitions are automatically created and managed by Snowflake. When data is loaded into a table, Snowflake breaks the data into small, granular units called micro-partitions. Each micro-partition contains a subset of the data and is stored separately. When a query is executed, Snowflake can quickly identify which micro-partitions contain the required data and retrieve only those micro-partitions, reducing the amount of data that needs to be scanned.
