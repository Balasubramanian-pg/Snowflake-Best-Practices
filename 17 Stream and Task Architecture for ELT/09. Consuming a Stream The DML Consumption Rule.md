# Consuming a Stream The DML Consumption Rule
Consuming a stream is a crucial aspect of stream processing, and the DML consumption rule plays a significant role in it. The DML consumption rule is used to consume a stream and apply DML (Data Manipulation Language) operations such as insert, update, and delete. This rule is essential in stream processing as it allows for real-time data processing and manipulation. The DML consumption rule can be used in various scenarios, including data integration, data transformation, and data aggregation.

## Implementation Example
```sql
CREATE STREAM orders_stream (order_id INT, customer_id INT, order_total DECIMAL(10, 2));
CREATE TABLE orders_table (order_id INT, customer_id INT, order_total DECIMAL(10, 2));

INSERT INTO orders_table
SELECT order_id, customer_id, order_total
FROM orders_stream;
```
Few examples and alternatives of SQL Code include using the `MERGE` statement instead of `INSERT` for upsert operations, or using the `UPDATE` statement for updating existing records.

## Explanation of the Code
Here's a procedural decomposition of the code:
* The `CREATE STREAM` statement is used to create a stream that captures the order data.
* The `CREATE TABLE` statement is used to create a table that will store the order data.
* The `INSERT INTO` statement is used to consume the stream and insert the data into the table.
* The `SELECT` statement is used to select the data from the stream.
* The `FROM` clause is used to specify the stream as the source of the data.
* The `orders_stream` and `orders_table` are used as the source and target of the data, respectively.
* The data types of the columns in the stream and table are matched to ensure compatibility.

## When to use 
The DML consumption rule can be used in the following scenarios:
* **Real-time data integration**: When integrating data from multiple sources in real-time, the DML consumption rule can be used to consume the streams and apply DML operations.
* **Data transformation**: When transforming data from one format to another, the DML consumption rule can be used to consume the stream and apply transformation operations.

## When not to use 
The DML consumption rule should not be used in the following scenarios:
* **Batch processing**: When processing large batches of data, the DML consumption rule may not be the most efficient approach.
* **Complex data processing**: When complex data processing is required, the DML consumption rule may not be sufficient, and additional processing may be required.

## Common Mistakes
Here are some common mistakes to avoid:
* Not matching the data types of the columns in the stream and table.
* Not handling errors and exceptions properly.
* Not optimizing the query for performance.
* Not considering the implications of consuming a stream on the underlying system.

## Expected Output
The expected output of the DML consumption rule is a table with the inserted data. For example:
| order_id | customer_id | order_total |
| --- | --- | --- |
| 1 | 1 | 100.00 |
| 2 | 2 | 200.00 |
| 3 | 3 | 300.00 |
The output will depend on the data in the stream and the DML operations applied.
