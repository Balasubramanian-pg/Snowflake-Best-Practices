# Applying Tags to Users and Roles in Snowflake

**Description:**
This topic explains how to apply tags to users and roles in Snowflake, enabling better organization, filtering, and access control. Tags can be used to categorize users and roles based on departments, teams, or other relevant criteria. This feature helps administrators manage access and permissions more efficiently.

## Short Explanation
**Overview:** Applying tags to users and roles allows for more granular access control and organization in Snowflake.

**Problem Solved:** Managing access and permissions for a large number of users and roles can be challenging. Tags help solve this problem by providing a flexible way to categorize and filter users and roles.

**Importance:** Tags are important in analytics and warehousing as they enable administrators to enforce fine-grained access control, reduce administrative overhead, and improve compliance.

**Use Cases:**
- Assigning tags to users based on their department or team to restrict access to sensitive data
- Using tags to categorize roles based on their permissions and access levels

### Typical Scenario
A company has multiple departments, each with its own set of users and roles. The administrator wants to restrict access to sensitive data based on departmental affiliations. They can apply tags to users and roles to achieve this.

### Core Snowflake SQL Objects Used
`ALTER USER`, `ALTER ROLE`

### Code Source Execution
```sql
ALTER USER 'user1' SET TAG department = 'Sales';
ALTER ROLE 'role1' SET TAG access_level = 'admin';
```
