# Rolling Back DML Operations via Suspend Immediate in Snowflake

**Description:**
This topic discusses how to roll back Data Manipulation Language (DML) operations in Snowflake using the Suspend Immediate feature. It allows users to quickly pause and rollback ongoing transactions. This feature is useful for handling unexpected issues during data modifications.

## Short Explanation
**Overview:** Suspend Immediate allows users to pause and rollback DML operations in Snowflake.

**Problem Solved:** Handling unexpected issues during data modifications by quickly rolling back changes.

**Importance:** Ensures data consistency and prevents data loss in case of errors or issues during DML operations.

**Use Cases:**
- Recovering from errors during data loading
- Rolling back changes during ETL processes

### Typical Scenario
An ETL process is loading large volumes of data into a Snowflake table. Due to data quality issues, the ETL process needs to be stopped and changes rolled back to maintain data consistency.

### Example Execution Logic Step-by-Step
To enable Suspend Immediate, set the STATEMENT_QUEUED_TIMEOUT_IN_SECONDS and STATEMENT_TIMEOUT_IN_SECONDS variables. Then, use the SUSPEND IMMEDIATE statement to pause and rollback a DML operation.

### Implementation Example
ALTER SESSION SET STATEMENT_QUEUED_TIMEOUT_IN_SECONDS = 300; ALTER SESSION SET STATEMENT_TIMEOUT_IN_SECONDS = 300; BEGIN TRANSACTION; INSERT INTO mytable (id, name) VALUES (1, 'John'); SUSPEND IMMEDIATE; ROLLBACK;

- Set STATEMENT_QUEUED_TIMEOUT_IN_SECONDS and STATEMENT_TIMEOUT_IN_SECONDS to enable Suspend Immediate.
