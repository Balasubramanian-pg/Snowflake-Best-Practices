# Monitoring Organization-level Storage via STORAGE_DAILY_HISTORY

**Description:**
This topic explains how to monitor organization-level storage usage in Snowflake using the STORAGE_DAILY_HISTORY view. It provides insights into daily storage usage trends, helping organizations optimize their storage costs. The STORAGE_DAILY_HISTORY view contains data about storage usage for the past 365 days.

A business intelligence team wants to monitor daily storage usage across their Snowflake organization to identify trends and optimize storage costs. They use the STORAGE_DAILY_HISTORY view to analyze storage usage over time.

`STORAGE_DAILY_HISTORY`

### Code Source Execution
```sql
SELECT * FROM STORAGE_DAILY_HISTORY;
```

### Example Execution Logic Step-by-Step
The STORAGE_DAILY_HISTORY view contains the following columns: DATE, STORAGE_USAGE. The DATE column represents the date for which the storage usage is reported, and the STORAGE_USAGE column represents the total storage usage for that date.

### Implementation Example
To monitor daily storage usage for the past 30 days, you can use the following query:
SELECT * FROM STORAGE_DAILY_HISTORY WHERE DATE >= DATE_TRUNC('day', CURRENT_DATE - INTERVAL '30 days');

### Explanation of the Code
- Step 1: The query selects all columns from the STORAGE_DAILY_HISTORY view.
- Step 2: The WHERE clause filters the data to include only the past 30 days of storage usage.
