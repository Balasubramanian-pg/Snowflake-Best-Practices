# Optimizing Join vs Scan Heavy Workloads with Query Profiler in Snowflake

**Description:**
This topic explains how to utilize Snowflake's Query Profiler to analyze and optimize join vs scan heavy workloads, identifying performance bottlenecks and providing actionable insights for improvement.

## Short Explanation
**Overview:** Snowflake's Query Profiler helps analyze query performance, enabling optimization of resource-intensive workloads.

**Problem Solved:** Identifying and optimizing performance bottlenecks in join and scan heavy workloads.

**Importance:** Crucial for maintaining efficient analytics and warehousing operations.

### Typical Scenario
A data analyst notices that a critical dashboard query is taking an excessively long time to execute, prompting an investigation into the query's performance using Snowflake's Query Profiler.

### Core Snowflake SQL Objects Used
`EXPLAIN`, `SHOW PROFILE`

### Code Source Execution
```sql
EXPLAIN (SELECT * FROM orders JOIN customers ON orders.customer_id = customers.customer_id);
```

### Example Execution Logic Step-by-Step
The EXPLAIN statement provides a high-level overview of the query plan, while the SHOW PROFILE command offers detailed insights into the query's execution, including step-by-step operator execution times and resource utilization.

### Implementation Example
To analyze a query's performance:
1. Enable query profiling: ALTER SESSION SET QUERY_PROFILE_LOAD_OVERVIEW = TRUE;
2. Execute the query with EXPLAIN: EXPLAIN (SELECT * FROM orders JOIN customers ON orders.customer_id = customers.customer_id);
3. Retrieve the query profile: SHOW PROFILE FOR QUERY <query_id> IN SESSION <session_id>;

### Explanation of the Code
- Step 1: Enable query profiling to collect detailed execution metrics.
- Step 2: Use EXPLAIN to obtain a high-level query plan overview.
- Step 3: Leverage SHOW PROFILE to access in-depth query execution details.
