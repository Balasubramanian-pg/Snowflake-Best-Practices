# What is a Snowflake Task?

**Description:**
A Snowflake Task is a SQL object that allows you to execute a SQL statement or a stored procedure on a scheduled basis. It enables automation of tasks, such as data loading, data transformation, and data validation. Snowflake Tasks are useful for streamlining data pipelines and ensuring data freshness.

## Short Explanation
**Overview:** Snowflake Task is a feature that allows you to schedule SQL statements or stored procedures to run at a specified interval.

**Problem Solved:** It solves the problem of automating repetitive data tasks, reducing manual effort, and improving data timeliness.

**Importance:** It matters in analytics and warehousing as it enables data teams to focus on higher-value tasks and ensures data consistency.

**Use Cases:**
- Automating daily data loads
- Running weekly data validation checks
- Executing monthly data summarization tasks

### Typical Scenario
A business scenario where a company wants to automate the daily loading of sales data from an external source into their Snowflake data warehouse. They can create a Snowflake Task to execute a SQL statement that loads the data on a daily basis.

### Core Snowflake SQL Objects Used
`CREATE TASK`

### Code Source Execution
```sql
CREATE TASK my_task
  WAREHOUSE = 'my_warehouse'
  SCHEDULE = '0 8 * * *'
  COMMENT = 'Daily data load task'
  AS
  CALL my_stored_procedure();
```

### Example Execution Logic Step-by-Step
The CREATE TASK statement is used to create a new Snowflake Task. The WAREHOUSE parameter specifies the warehouse to use for the task, the SCHEDULE parameter specifies the schedule for the task, and the COMMENT parameter provides a description of the task. The AS clause specifies the SQL statement or stored procedure to execute.
