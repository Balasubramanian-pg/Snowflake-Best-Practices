# Monitoring Task Executions via TASK_HISTORY in Snowflake

**Description:**
This topic explains how to monitor task executions in Snowflake using the TASK_HISTORY table. It provides insights into task performance, errors, and execution history. By analyzing TASK_HISTORY, users can optimize and troubleshoot task workflows.

**Overview:** TASK_HISTORY provides a record of task executions, including execution time, status, and error messages.

- Optimizing task performance
- Auditing task executions

### Typical Scenario
A data engineer wants to monitor the execution history of a task that loads data from a source table to a target table. They use TASK_HISTORY to analyze execution times, errors, and task dependencies.

### Core Snowflake SQL Objects Used
`TASK`, `TASK_HISTORY`

### Code Source Execution
```sql
SELECT * FROM TABLE(INFORMATION_SCHEMA.TASK_HISTORY_BY_SESSION());
```

### Example Execution Logic Step-by-Step
The TASK_HISTORY table provides a record of task executions, including:
    - TASK_ID: The unique identifier for the task.
    - SESSION_ID: The session ID that executed the task.
    - EXECUTION_START_TIME: The start time of the task execution.
    - EXECUTION_END_TIME: The end time of the task execution.
    - STATUS: The status of the task execution (e.g., 'SUCCEEDED', 'FAILED').
    - ERROR_MESSAGE: The error message if the task failed.

### Implementation Example
