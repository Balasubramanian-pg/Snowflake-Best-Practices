# Suspending and Resuming Tasks in Snowflake

**Description:**
This topic explains how to suspend and resume tasks in Snowflake, which is useful for managing and maintaining task performance and resource utilization.

## Short Explanation
**Overview:** Tasks in Snowflake can be suspended and resumed to manage performance and resources.

**Problem Solved:** Suspending and resuming tasks helps prevent overloading and optimize resource usage.

**Importance:** This feature is important for maintaining efficient data pipelines and workflows.

**Use Cases:**
- Data pipeline maintenance
- Resource optimization

### Typical Scenario
A business scenario where a data pipeline needs to be paused during maintenance and resumed afterwards.

### Core Snowflake SQL Objects Used
`ALTER TASK`

### Code Source Execution
```sql
ALTER TASK my_task SUSPEND;
```

### Example Execution Logic Step-by-Step
To suspend a task, use the ALTER TASK command with the SUSPEND option. To resume a task, use the ALTER TASK command with the RESUME option.

### Implementation Example
CREATE TASK my_task
  WAREHOUSE = 'my_warehouse'
  SCHEDULE = '0 12 * * *'
  COMMENT = 'My task'
  CALL my_procedure();

ALTER TASK my_task SUSPEND;

ALTER TASK my_task RESUME;

### Explanation of the Code
- Step 1: Create a task with a schedule and procedure call.
- Step 2: Suspend the task using ALTER TASK with SUSPEND option.
- Step 3: Resume the task using ALTER TASK with RESUME option.

### When to Use
- During maintenance
- For resource optimization

### When Not to Use
- When tasks need to run continuously
- When tasks are critical and cannot be paused

### Common Mistakes
- Forgetting to resume tasks after suspension
- Suspending critical tasks
