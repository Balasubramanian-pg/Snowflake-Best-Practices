# Applying Tags to Databases and Schemas in Snowflake

**Description:**
This topic explains how to apply tags to databases and schemas in Snowflake, enabling better organization, cost allocation, and chargeback. Tags help categorize and track Snowflake resources for more efficient management. By applying tags, users can easily filter and report on their Snowflake resources.

## Short Explanation
**Overview:** Applying tags to databases and schemas allows for better resource management and cost allocation in Snowflake.

**Problem Solved:** Tags solve the problem of disorganized and untracked Snowflake resources, making it difficult to manage costs and allocate charges.

**Importance:** Tags are important in analytics and warehousing as they enable efficient resource management, cost allocation, and chargeback.

**Use Cases:**
- Organizing databases and schemas by department or project
- Tracking costs and allocating charges to specific business units

### Typical Scenario
A company has multiple departments using Snowflake, and they want to track costs and allocate charges to specific departments. They can apply tags to databases and schemas to achieve this.

### Core Snowflake SQL Objects Used
`ALTER DATABASE`, `ALTER SCHEMA`

### Code Source Execution
```sql
ALTER DATABASE my_database SET TAG department = 'Sales';
ALTER SCHEMA my_schema SET TAG project = 'Marketing';
```

### Example Execution Logic Step-by-Step
To apply a tag to a database or schema, use the ALTER statement with the SET TAG clause. For example, to apply a tag named 'department' with the value 'Sales' to a database named 'my_database', use the following SQL statement: ALTER DATABASE my_database SET TAG department = 'Sales';

### Implementation Example
CREATE DATABASE my_database;
ALTER DATABASE my_database SET TAG department = 'Sales';
CREATE SCHEMA my_schema;
ALTER SCHEMA my_schema SET TAG project = 'Marketing';

### When to Use
- When you need to track costs and allocate charges to specific departments or projects
- When you want to organize your Snowflake resources by category or attribute

### When Not to Use
