# Identifying Primary and Foreign Keys via Table Constraints in Snowflake

**Description:**
This topic explains how to identify primary and foreign keys in Snowflake using table constraints. Table constraints are used to define the relationships between tables in a database. Understanding primary and foreign keys is crucial in maintaining data consistency and performing joins.

## Short Explanation
**Overview:** Primary keys uniquely identify each row in a table, while foreign keys establish relationships between tables.

**Problem Solved:** Identifying primary and foreign keys helps in maintaining data integrity and enables efficient querying.

**Importance:** Primary and foreign keys are essential in analytics and warehousing as they enable data consistency and efficient data retrieval.

**Use Cases:**
- Data modeling

- Business intelligence

### Typical Scenario
A company has an e-commerce platform with a database containing customer information, orders, and products. The customer ID is a primary key in the customer table, and the customer ID in the orders table is a foreign key referencing the customer table.

### Core Snowflake SQL Objects Used
`INFORMATION_SCHEMA.TABLE_CONSTRAINTS`, `INFORMATION_SCHEMA.KEY_COLUMN_USAGE`

### Code Source Execution
```sql
SELECT * FROM INFORMATION_SCHEMA.TABLE_CONSTRAINTS WHERE CONSTRAINT_TYPE IN ('PRIMARY KEY', 'FOREIGN KEY');
```

### Example Execution Logic Step-by-Step
The INFORMATION_SCHEMA.TABLE_CONSTRAINTS table contains information about table constraints, including primary keys and foreign keys. The CONSTRAINT_TYPE column indicates the type of constraint, and the TABLE_NAME and COLUMN_NAME columns provide information about the table and column involved in the constraint.

### Implementation Example
To identify primary and foreign keys in a Snowflake database, you can use the following SQL query:

SELECT 
  TABLE_NAME, 
  COLUMN_NAME, 
  CONSTRAINT_NAME, 
  CONSTRAINT_TYPE
FROM 
  INFORMATION_SCHEMA.KEY_COLUMN_USAGE
WHERE 
  TABLE_NAME IN ('CUSTOMERS', 'ORDERS', 'PRODUCTS');

### Explanation of the Code
- The query selects the table name, column name, constraint name, and constraint type from the INFORMATION_SCHEMA.KEY_COLUMN_USAGE table.
- The WHERE clause filters the results to only include tables of interest, such as 'CUSTOMERS', 'ORDERS', and 'PRODUCTS'.
