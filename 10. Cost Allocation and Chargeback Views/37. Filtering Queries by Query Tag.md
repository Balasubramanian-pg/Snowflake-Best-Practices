# Filtering Queries by Query Tag in Snowflake

**Description:**
This feature allows Snowflake administrators to filter and manage queries based on query tags, making it easier to organize, monitor, and optimize queries. Query tags are labels that can be assigned to queries to categorize and track them. By filtering queries by query tag, administrators can quickly identify and manage queries that match specific criteria.

## Short Explanation
**Overview:** Filtering queries by query tag enables efficient query management and optimization.

**Problem Solved:** Query management and organization become challenging as the number of queries increases.

**Importance:** This feature is essential for large-scale Snowflake deployments where query management is critical.

**Use Cases:**
- Filtering queries by department
- Identifying queries by project
- Monitoring queries by data sensitivity

### Typical Scenario
A Snowflake administrator wants to monitor and optimize queries related to a specific department. They can assign a query tag to these queries and then filter them to view and manage them separately.

### Core Snowflake SQL Objects Used
`ALTER SESSION`, `SHOW QUERIES`

### Code Source Execution
```sql
ALTER SESSION SET QUERY_TAG = 'department: sales';
SHOW QUERIES TAG 'department: sales';
```

### Example Execution Logic Step-by-Step
In this example, we first set the query tag for the current session to 'department: sales' using the ALTER SESSION command. Then, we use the SHOW QUERIES command with the TAG option to filter queries that have the 'department: sales' tag.
